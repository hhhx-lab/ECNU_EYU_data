# 项目规则

## 提交规范

1. 每次提交成果的 Git commit message 必须使用简体中文，简单描述改动内容。
2. 提交的代码需要具有可读性，必要时补充注释或文档说明。
3. 提交上传的文档必须可读，不要出现乱码；公式不要以 LaTeX 源码形式呈现在最终文档中。
4. 不要改动其他同学工作区的内容。
5. 不要提交冗余文件，例如缓存、临时测试文件、调试产物、本地代理配置等。
6. 提交改动尽量保持一周两次左右，不低于两次。

## 工作区边界

- `work_space/S1`、`work_space/S2`、`work_space/S3`、`work_space/S4`、`work_space/G1`、`work_space/G2` 分别对应不同成员或小组。
- 每个成员只在自己的工作区内提交代码、数据说明、文档和结果文件。
- 如需使用公共资料，请放入或引用 `data_space`，不要复制多份难以同步的材料。
- 阶段进展统一更新到 `time_pipeline/README.md`。
