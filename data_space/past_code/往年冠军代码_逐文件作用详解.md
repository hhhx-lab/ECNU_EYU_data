# 往年冠军代码逐文件作用详解

本文逐文件说明 `往年文章和code/2025` 与 `往年文章和code/2023` 中的核心代码。由于 `2023/Segmentation_Tasks/nnUNet_install` 和 `2023/Segmentation_Tasks/mednext` 是完整外部框架源码，数量很大，本文对作者核心脚本逐文件解释，对外部框架按模块解释；真正需要我们理解和改造的文件主要集中在 `GliGAN`、各任务 `*_inference`、`Synthesis/Task7`、`Synthesis/Task8`。

## 2025 BraTS Orchestrator

### 根目录文件

| 文件 | 作用 |
| --- | --- |
| `2025/README.md` | 项目总说明，介绍可用任务、算法枚举、安装方式、Docker/Singularity 依赖和调用示例。 |
| `2025/pyproject.toml` | Python 包配置。定义包名 `brats`、依赖项 `docker`、`spython`、`rich`、`PyYAML`、`dacite`、`nibabel` 等，并声明 `preprocessing` 可选依赖。 |
| `2025/poetry.lock` | Poetry 锁文件，固定依赖版本。 |
| `2025/CITATION.cff` | 引用信息。 |
| `2025/LICENSE` | Apache 2.0 许可证。 |
| `2025/CONTRIBUTING.md` | 贡献指南。 |
| `2025/.readthedocs.yaml` | ReadTheDocs 文档构建配置。 |
| `2025/2025年第一名.pdf` | 本地保存的 2025 第一名相关论文/材料，不是可执行代码。 |

### `brats` 包入口

| 文件 | 作用 |
| --- | --- |
| `2025/brats/__init__.py` | 包的公开入口。导出 `Inpainter`、`MissingMRI`、各类 `Segmenter`，并默认移除 loguru handler，避免包导入时自动输出日志。 |
| `2025/brats/constants.py` | 全局常量中心。定义 `Backends`、`Task`、所有算法枚举、数据目录、元数据 YAML 路径、参数目录、Zenodo API、输出命名模板和包引用链接。 |
| `2025/brats/preprocessing.py` | 预处理封装。调用 `brainles_preprocessing` 做配准、去颅骨/defacing、归一化，并根据 challenge 类型自动选择 SRI24/MNI152、是否允许缺失模态等流程。 |

### `brats/core`

| 文件 | 作用 |
| --- | --- |
| `2025/brats/core/__init__.py` | 空包标记文件。 |
| `2025/brats/core/brats_algorithm.py` | 所有算法类的抽象父类。负责加载算法 YAML、保存算法元数据、创建临时输入/输出目录、标准化输入、选择 Docker/Singularity 后端、处理单病例和批量输出。 |
| `2025/brats/core/segmentation_algorithms.py` | 分割任务类。定义四模态分割基类、成人胶质瘤/非治疗后胶质瘤/脑膜瘤/儿童/Africa/MET/GoAT 等具体 Segmenter；另有 `MeningiomaRTSegmenter` 只接收 T1c。 |
| `2025/brats/core/inpainting_algorithms.py` | Inpainting 任务类。输入 `t1n-voided` 和 `mask`，复制成算法容器要求的文件名，调用统一推理父类。 |
| `2025/brats/core/missing_mri_algorithms.py` | 缺失模态生成任务类。单病例和批量都要求恰好提供 3 个 MRI 模态，由容器生成缺失的第 4 个。 |
| `2025/brats/core/docker.py` | Docker 后端。负责连接 Docker daemon、拉镜像、判断 CUDA、设置 GPU/CPU、挂载输入/输出/参数/额外文件目录、运行容器、监听日志、检查输出。 |
| `2025/brats/core/singularity.py` | Singularity 后端。把 Docker 镜像转换成 Singularity sandbox，设置 bind mount、overlay、`--nv` GPU 选项，并复用 Docker 后端的部分配置逻辑。 |

### `brats/utils`

| 文件 | 作用 |
| --- | --- |
| `2025/brats/utils/__init__.py` | 空包标记文件。 |
| `2025/brats/utils/algorithm_config.py` | YAML 配置解析。定义 `MetaData`、`RunArgs`、`AdditionalFilesData`、`AlgorithmData` 等 dataclass，用 `yaml.safe_load` 和 `dacite.from_dict` 把 YAML 转成强类型对象。 |
| `2025/brats/utils/data_handling.py` | 推理文件管理。包含临时目录 context manager、日志文件 handler、临时目录删除、输入 NIfTI shape sanity check。 |
| `2025/brats/utils/exceptions.py` | 自定义异常。包括 CPU 不兼容、容器运行失败、算法配置错误、Zenodo 访问错误。 |
| `2025/brats/utils/logging.py` | 日志控制。提供禁用/启用包日志、添加/移除控制台 handler，并用锁保证 console handler 单例。 |
| `2025/brats/utils/zenodo.py` | 额外权重/文件下载。根据 Zenodo record id 查询最新版本，下载 archive，解压到 `brats/data/additional_files`，并处理本地缓存。 |

| `brats24_missing_mri_cgv.yml` | 2024 missing MRI CGV 参数，含 dataset、dataloader、model、refiner、discriminator、training 等完整配置。 |

## GliGAN 合成数据模块

### `GliGAN/src/train`

| 文件 | 作用 |
| --- | --- |
| `csv_creator.py` | 为 GliGAN 训练生成 CSV。遍历训练集，找到四模态和 `seg`，计算肿瘤中心、三轴边界框、尺寸，只保留三轴尺寸都不超过 96 的病例，并保存模态路径和坐标信息。 |
| `tumour_main.py` | 每个 MRI 模态的局部肿瘤影像生成器训练脚本。生成器可选 SwinUNETR、AttentionUnet、UNet；判别器来自 `Discriminator.py`；输入为 noisy crop + label crop，输出重建 crop。支持 GAN 损失、L1 重建损失、两阶段训练、resume、保存 checkpoint 和 NIfTI 样例。 |
| `tumour_main_baseline.py` | 模态生成器 baseline。只用 L1 重建损失训练 SwinUNETR，没有判别器，主要作为 GliGAN 对照实验。 |
| `label_main.py` | 肿瘤标签生成器训练脚本。使用 alpha-GAN 风格的 Generator/Encoder/Discriminator/Code Discriminator，从 cropped label 学习生成 3D 肿瘤形状。保存 G/D/E/CD 四组权重。 |

### `GliGAN/src/networks`

| 文件 | 作用 |
| --- | --- |
| `Discriminator.py` | 模态生成器使用的 3D 判别器。输入通道是重建影像和标签拼接后的张量，使用 spectral norm Conv3d 下采样，最后输出真假判别。 |
| `LabelGAN.py` | 标签生成器网络定义。包含 3D Discriminator、Code_Discriminator、Encoder、Generator。Generator 从 latent vector 上采样生成 64 立方体标签块。 |
| `__init__.py` | 空包标记文件。 |

### `GliGAN/src/utils`

| 文件 | 作用 |
| --- | --- |
| `data_utils.py` | GliGAN 训练 DataLoader。根据模态选择 CSV 列，选择不同任务的 label transform，加载 NIfTI，做 label 多通道转换、局部 crop、噪声注入、CacheDataset 和 DataLoader。 |
| `convert_to_multi_channel_based_on_brats_classes.py` | 标签转区域通道。支持 BraTS 2023/GoAT 的 TC/WT/ET、BraTS 2024 glioma 的 TC/WT/ET/RC、Meningioma 的 GTV。 |
| `crop_label.py` | 根据 CSV 中的肿瘤边界框裁剪 label，并 pad 到 `96x96x96`。主要给 label generator 训练使用。 |
| `gaussian_noise_tumour.py` | 只在肿瘤 mask 内加高斯噪声，用于让生成器学习从噪声恢复局部肿瘤影像。 |
| `gaussian_noise_tumour_extended.py` | 在肿瘤及其周围球形区域加噪声，比只在肿瘤区加噪声更难、更接近边界重建。 |
| `__init__.py` | 空包标记文件。 |

### `GliGAN/src/infer`

| 文件 | 作用 |
| --- | --- |
| `main_random_label_random_dataset_generator_multiprocess.py` | 胶质瘤/GoAT 合成病例生成脚本。加载四个模态生成器和 label generator，随机选择或生成肿瘤标签，寻找脑内可放置位置，生成各模态局部 patch，融合回原图，保存新的合成病例。支持用 `start_case/end_case` 分片并行。 |

## 外部框架源码边界

### `Segmentation_Tasks/nnUNet_install`

这是嵌入的 nnUNet v2 源码。冠军代码依赖它进行：

- 数据格式检查和预处理。
- 训练标准 nnUNet、Swin UNETR 变体和 2021 winner 风格 trainer。
- 执行 `nnUNetv2_plan_and_preprocess`、`nnUNetv2_train`、`nnUNetv2_predict`、`nnUNetv2_ensemble`。

主要模块含义：

| 目录/文件组 | 作用 |
| --- | --- |
| `nnunetv2/dataset_conversion/` | 各类公开数据集转换脚本和 `generate_dataset_json.py`。本项目实际主要参考 notebook 和 example JSON。 |
| `nnunetv2/experiment_planning/` | 根据数据 spacing/shape 生成 plans。 |
| `nnunetv2/preprocessing/` | 裁剪、归一化、重采样、预处理数据保存。 |
| `nnunetv2/training/` | trainer、loss、数据增强、网络训练循环。 |
| `nnunetv2/inference/` | 滑窗推理、概率图保存、ensemble 输入输出。 |
| `nnunetv2/evaluation/` | Dice 等评估与最佳配置选择。 |
| `nnunetv2/utilities/` | 文件名、路径、label handling、plans handling 等工具。 |
| `documentation/` | nnUNet 官方文档。 |
| `setup.py` | 本地 editable install 入口。 |

一般不建议先改 nnUNet 框架源码。更稳的做法是新增 trainer、plans 或在外层推理脚本处理输入输出。

## 代码阅读优先级

如果目标是理解“冠军为什么强”，优先级如下：

1. `2023/Segmentation_Tasks/BraTS2023_inference/infer_low_disk.py`
   直接体现提交推理、集成和后处理。

2. `2023/Segmentation_Tasks/GliGAN/src/train/csv_creator.py`
   理解他们如何把肿瘤变成可生成的局部 patch 数据。

3. `2023/Segmentation_Tasks/GliGAN/src/train/tumour_main.py`
   理解每个模态生成器如何训练。

4. `2023/Segmentation_Tasks/GliGAN/src/train/label_main.py`
   理解合成肿瘤标签从何而来。

5. `2023/Segmentation_Tasks/GliGAN/src/infer/main_random_label_random_dataset_generator_multiprocess.py`
   理解如何把合成标签和合成影像变成新的完整训练病例。

6. `2025/brats/core/brats_algorithm.py` 与 `2025/brats/core/docker.py`
   理解如何把算法封装成可复用推理 API。

7. `2025/brats/data/meta/metastases.yml`
   对 2026 脑转移瘤任务最相关，能看到 MET 任务历史容器注册方式。

## 移植到 2026 Task 1 时最该改的位置

如果要基于这些代码思路搭 2026 BraTS-MET 项目，不建议直接在原目录里硬改。建议新建自己的项目目录，并借鉴这些位置：

| 需求 | 可参考文件 |
| --- | --- |
| 官方数据转 nnUNet 格式 | `BraTS2023_inference/infer_low_disk.py` 的 `convert_case_to_nnUNet()` |
| 多模型推理命令组织 | 各 `*_inference/infer_low_disk*.py` 的 `perform_inference_step()` |
| 低磁盘概率图集成 | `ensemble_predictions()` |
| 小假阳性清理 | `filter_structures_by_volume()`、`get_connected_components()`、`thresholding_step()` |
| 标签/区域映射 | `convert_to_multi_channel_based_on_brats_classes.py` 和各 infer 文件中的 `convert_labels_back_to_BraTS()` |
| 推理封装成 API | `2025/brats/core/brats_algorithm.py` |
| 算法元数据配置化 | `2025/brats/data/meta/*.yml` + `algorithm_config.py` |

最不能直接照搬的是标签定义和阈值。BraTS-MET 的病灶分布、标签含义、评价指标和 2023/2024 glioma 不同，需要重新统计训练集病灶体积，再决定后处理策略。
