## 3. 编写冒烟测试 SLURM 脚本

创建 `/public/home/zqchen/Code/run_gligan_tumour_test.slurm`：

```bash
cat > /public/home/zqchen/Code/run_gligan_tumour_test.slurm << 'EOF'
#!/bin/bash
#SBATCH -J gligan_test
#SBATCH -p a100
#SBATCH -N 1
#SBATCH -n 1
#SBATCH --gres=gpu:1
#SBATCH --cpus-per-task=4
#SBATCH --mem=32G
#SBATCH -t 00:30:00
#SBATCH -o %x-%j.out
#SBATCH -e %x-%j.err

module purge
module load apps/envs/miniconda3/25.5.1
module load compiler/cuda/12.1

source activate diffusion

cd /public/home/zqchen/Code/Segmentation_Tasks/GliGAN/src/train

echo "========== 环境检查 =========="
python -c "import torch; print('CUDA:', torch.cuda.is_available()); print('Device:', torch.cuda.get_device_name(0))"
nvidia-smi

echo "========== 开始冒烟测试 =========="

python tumour_main_diffusion.py \
    --logdir brats2024_tumour \
    --dataset BRATS_2024 \
    --modality t1ce \
    --generator_type Unet \
    --batch_size 2 \
    --num_workers 2 \
    --in_channels 5 \
    --out_channels 1 \
    --n_steps 1000 \
    --beta_schedule cosine \
    --num_steps 10 \
    --optim_lr 2e-4 \
    --noise_type gaussian_tumour

echo "========== 冒烟测试完成 =========="
EOF
```

> 关键：`--num_steps 10`（只跑 10 步），`-t 00:30:00`（作业限时 30 分钟），`--generator_type Unet`（轻量 backbone）。跑通后确认无误，再改为正式训练参数。

## 5. 冒烟测试通过的标准

查看 `gligan_test-JOBID.out`，确认以下内容：

1. **环境检查通过**：`CUDA: True`，显示 `NVIDIA A100`
2. **数据加载正常**：没有 `FileNotFoundError` 或 `No such file`
3. **训练启动**：看到类似 `Step:1/10, Loss:xxx, Time:xxx` 的输出
4. **Loss 在变化**：10 步中 loss 不是 NaN，且呈下降趋势
5. **最终输出**：`========== 冒烟测试完成 ==========`

如果 10 步全部跑完没有报错，说明代码和环境完全兼容。

## 7. 常见问题

| 现象 | 可能原因 | 解决 |
|------|---------|------|
| `--test-only` 报 rejected | 分区名写错或 GPU 型号不对 | 用 `sinfo` 看可用分区，确认是 `a100` 还是 `v100` |
| CSV 0 行 | 肿瘤包围盒 > 96 | 按第 2 节改过滤条件 |
| CUDA out of memory | batch_size 太大 | 降到 `--batch_size 4` 或 `2` |
| `conda: command not found` | module 没加载 | 先执行 `module load apps/envs/miniconda3/25.5.1` |
| `ModuleNotFoundError: No module named 'xxx'` | 依赖漏装 | 回到第 1 节补装缺失的包 |
| 任何 `pip install` 触发 numpy 2.x 源码编译 | pip 构建隔离环境依赖 `numpy>=2.0.0`，集群 glibc 太旧 | 把触发编译的包加到 conda 安装列表（预编译二进制），例如 `conda install -y scikit-image nilearn pywavelets` |
| 作业一直 PD 不运行 | GPU 资源被占满 | `gpus` 查看空闲情况，等待或换 `v100` 分区 |
| Loss 全是 NaN | 学习率太高或数据异常 | 降低 `--optim_lr 1e-5`，检查 CSV 中文件路径是否可达 |
| 输出文件乱码 | 登录节点 locale 警告 | 忽略，不影响 SLURM 作业 |
