> 安全说明：本文档不保存账号密码。账号密码应只在登录时输入，或使用 SSH key 管理，不要写入脚本、仓库、Markdown、Notebook 或命令历史。

## 1. 核心结论

这是一个标准的 SSH 登录加 SLURM 调度的高性能计算集群。

登录节点只用于：

- 编辑脚本
- 上传下载文件
- 管理项目目录
- 查看资源状态
- 提交、查看、取消 SLURM 作业

登录节点严禁用于：

- 直接跑训练任务
- 直接跑 GPU/CPU 密集程序
- 多线程压测
- 长时间编译
- 挖矿或恶意占用资源

所有正式计算任务都应该通过 `sbatch`、`srun` 或 `salloc` 交给 SLURM 调度系统运行。

```bash

```

从超算拉回结果：

```bash
rsync -avP -e "ssh -p 2323" zqchen@59.78.189.133:/public/home/zqchen/my_project/results/ ./results/
```


## 6. SLURM 基本流程

官方手册给出的作业流程可以概括为：

1. 编写作业脚本，声明作业名、分区、节点数、CPU/GPU、内存、时间等资源。
2. 使用 `sbatch` 提交给 SLURM。
3. 作业进入队列等待。
4. 调度器检查资源是否满足。
5. 满足条件后在计算节点运行。
6. 用 `squeue` 查看状态，发现错误可用 `scancel` 取消。
7. 作业结束后查看 `.out`、`.err`、日志和结果文件。

提交作业：

```bash
sbatch run.slurm
```

提交前做调度预测，不真正提交：

```bash
sbatch --test-only -p cpu_96G -N 1 -n 1 --cpus-per-task=1 --mem=1G --time=00:01:00 --wrap=hostname
sbatch --test-only -p a100 -N 1 -n 1 --gres=gpu:1 --cpus-per-task=1 --mem=4G --time=00:01:00 --wrap=nvidia-smi
```

实测 `cpu_96G`、`v100`、`a100` 的 `--test-only` 均可被调度器接受。

## 12. 外部 API 代理

官方说明：计算节点无法直接访问外部网络。如果计算作业中需要调用外部 API，可使用登录节点 API 代理服务。

适用场景：

- 大模型 API 代理
- PubChem 等外部 API 内网转发
- 计算节点作业中需要访问外部接口

绑定规则：

```text
绑定 IP：10.11.100.254
端口范围：5000-5010
端口由系统自动分配
推荐在 login02，即 59.78.189.133 上启动
```

启动前安装依赖时，应优先在个人环境里安装，不要污染公共环境：

```bash
module load apps/envs/miniconda3/25.5.1
conda create -n api_proxy python=3.11
conda activate api_proxy
pip install requests flask
```

启动一个普通目标代理：

```bash
python /public/software/profile.d/api_proxy.py \
  --target "目标接口地址" \
  --name "服务自定义名称"
```

代理 OpenAI 兼容接口示例：

```bash
python /public/software/profile.d/api_proxy.py \
  --target https://api.openai.com/v1 \
  --api-key "$OPENAI_API_KEY" \
  --auth-header "Authorization"
```

启动成功后，会输出类似：

```text
绑定地址：http://10.11.100.254:5000
```

计算节点作业中访问：

```bash
curl http://10.11.100.254:5000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-model",
    "messages": [{"role": "user", "content": "你好"}],
    "temperature": 0.7
  }'
```

后台运行：

```bash
nohup python /public/software/profile.d/api_proxy.py \
  --target http://example.com \
  --name "my-api-proxy" > proxy.log 2>&1 &
```

停止代理：

```bash
ps -ef | grep proxy_server.py | grep -v grep
kill -9 PID
```

API 代理注意事项：

- 只能访问 `http://10.11.100.254:端口`，不要用 `127.0.0.1` 或其他 IP。
- 端口只使用系统自动分配的 `5000-5010`。
- 用完及时关闭，避免占用端口和进程。
- 不要二次转发，不要扩大访问范围。
- 不要传输涉密或敏感数据。
- `--target` 填 API 根地址，避免重复拼接 `/v1` 导致 404。

## 14. 常见问题排查

查看当前账号有没有作业：

```bash
squeue -u zqchen
```

查看某个作业详情：

```bash
scontrol show job JOBID
```

查看作业历史和退出码：

```bash
sacct -j JOBID --format=JobID,JobName,Partition,State,ExitCode,Elapsed,MaxRSS
```

查看输出：

```bash
less jobname-JOBID.out
less jobname-JOBID.err
```

取消作业：

```bash
scancel JOBID
```

常见状态：

| 状态 | 含义 |
| --- | --- |
| `PD` | Pending，排队中 |
| `R` | Running，运行中 |
| `CG` | Completing，收尾中 |
| `CD` | Completed，完成 |
| `F` | Failed，失败 |
| `TO` | Timeout，超时 |
| `OOM` | 内存不足 |

常见原因：

- 分区名写错。
- 内存申请太少。
- 时间限制太短。
- `conda activate` 在 batch 脚本里没有生效。
- 在登录节点直接运行了计算程序。
- GPU 作业忘记写 `--gres=gpu:1`。
- 作业脚本使用了旧模板中的过期分区名。

1. 登录 `login2`。

