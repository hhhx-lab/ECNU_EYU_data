# G1 T2W 缺失模态填补数据准备说明

更新日期：2026-06-18

## 1. 当前 G1 新代码口径

新 G1 代码是缺失模态填补流程，不是局部 ROI synthetic augmentation 流程。当前目标是：

1. 用完整四模态病例训练 `t1n+t1c+t2f -> t2w`。
2. 对 T2W 被判定为 fake 或需要重建的病例，移除原 T2W 后放入 `data/input_inference/`。
3. 由 EncDec/BBDM/ensemble 在 `data/output/<case_id>/` 中生成新的 `<case_id>-t2w.nii.gz`，并保留 t1n/t1c/t2f/seg 软链接，供 G2 直接 QC。

## 2. 数据放置规则

训练目录：

```text
data/input/<case_id>/
  <case_id>-t1n.nii.gz
  <case_id>-t1c.nii.gz
  <case_id>-t2w.nii.gz
  <case_id>-t2f.nii.gz
  <case_id>-seg.nii.gz
```

推理目录：

```text
data/input_inference/<case_id>/
  <case_id>-t1n.nii.gz
  <case_id>-t1c.nii.gz
  <case_id>-t2f.nii.gz
  <case_id>-seg.nii.gz
```

注意：`input_inference` 中应故意没有 `t2w`。如果原始 T2W 是 fake/broken，要从该目录中去掉，而不是保留一个坏文件。

推理输出目录：

```text
data/output/<case_id>/
  <case_id>-t1n.nii.gz  # symlink to inference input
  <case_id>-t1c.nii.gz  # symlink to inference input
  <case_id>-t2f.nii.gz  # symlink to inference input
  <case_id>-seg.nii.gz  # symlink to inference input
  <case_id>-t2w.nii.gz  # generated by G1
```

## 3. 当前 G2 自动分配

根据 G2 现有结果：

1. `real_train_manifest.csv` 中 final QC pass 病例为 1295 例。
2. gzip header 判定 T2W fake 的病例为 265 例。
3. 因此默认分配为：
   - `data/input/`：1030 例完整真实 T2W 训练病例。
   - `data/input_inference/`：265 例待补 T2W 病例，T2W 文件不链接。

## 4. 生成数据目录

在新 G1 代码目录运行：

```bash
python prepare_g1_t2w_data.py --mode symlink --clean --overwrite
```

默认会读取：

```text
../../../../G2/results/manifests/real_train_manifest.csv
../../../../G2/results/qc/official_fake_t2w_cases_by_gzip_header_2026-06-15.csv
```

输出：

```text
data/input/
data/input_inference/
data/g1_data_placement_manifest.csv
```

## 5. preprocess 后统一验证集

先运行：

```bash
python preprocess.py
```

它会生成：

```text
data/data_csv.csv
data/latents/<case_id>/*.npy
```

然后运行：

```bash
python mark_val_split_from_g2.py
```

该脚本会按 G2 fixed fold0 把 CSV 中可用完整病例标记为 `train` 或 `val`，避免手工改错。

当前 G2 fixed fold0 对完整真实 T2W 训练集的投影是：

1. train：820 例。
2. val：210 例。

其余 49 个 fixed val 病例属于 fake T2W 队列，不进入 G1 完整 T2W 训练 CSV。

## 6. 后续命令顺序

```bash
python preprocess.py
python mark_val_split_from_g2.py
python generate_attmask.py
python compute_weights.py
```

`compute_weights.py` 输出的 `channel_importance_weights` 需要复制到 `training_bbdm.py` 的 `args_train`。

## 7. G2 对输出的接收要求

G1 生成完成后，G2 要检查：

1. `data/output/<case_id>/<case_id>-t2w.nii.gz` 是否覆盖全部 `data/input_inference` 病例。
2. 输出 shape/spacing/affine 是否与输入病例内其他模态一致。
3. 输出 T2W 是否存在 NaN/Inf、常数图、强度分布异常。
4. 输出补回原病例后，是否能重新形成四模态完整病例供 S1/S2 训练。
5. 对有真实 T2W 的 held-out complete val，使用 SSIM/PSNR/MSE/MAE 做模型选择。

## 8. G2 需要更新的旧口径

旧的 `G1_G2_diffusion_output_contract.md` 主要服务于 GliGAN-compatible synthetic augmentation。新 G1 代码交付前，G2 应新增或改写一份 T2W imputation contract，避免把缺失模态填补和 synthetic augmentation 混成一条链。
