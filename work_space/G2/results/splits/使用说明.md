# splits 使用说明

这个目录保存固定训练/验证划分。它的目的不是随机探索，而是给后续所有消融实验一个稳定、公平的验证基准。

## 文件总览

| 文件 | 内容 | 用途 |
|---|---|---|
| `splits_final_fold0_realval.json` | 1 个 fold，`train` 1036 例，`val` 259 例 | nnU-Net 和 real-only/real+synth 消融共用的固定真实验证 fold |

## `splits_final_fold0_realval.json`

这个 JSON 是 nnU-Net 常见 split 格式。最外层是一个列表，目前只有一个 fold。

结构如下：

```json
[
  {
    "train": ["BraTSMET_000001", "..."],
    "val": ["BraTSMET_000005", "..."]
  }
]
```

字段含义：

| 字段 | 含义 |
|---|---|
| `train` | real-only 训练用病例 ID，当前 1036 例 |
| `val` | 固定真实验证病例 ID，当前 259 例 |

这些 ID 使用 `nnunet_case_mapping_realonly.csv` 里的 `nnunet_case_id`，不是原始 BraTS ID。

## 为什么要固定这个 split

后续要比较至少三类实验：

1. real-only baseline。
2. real + 少量 accepted synthetic。
3. real + 不同生成策略或不同比例 synthetic。

如果每次随机切分不一样，就很难判断提升来自 synthetic data，还是来自验证集变化。因此这个 split 应该固定复用。

## 使用规则

1. 所有消融实验必须使用同一个 `val` 列表。
2. synthetic data 只能加入训练侧，不能加入 `val`。
3. 官方 validation 数据不在这个 split 里，因为它没有公开 label。
4. 如果确实要换 split，应新建 `splits_final_fold1_realval.json` 或类似文件，不要静默覆盖现有文件。
5. 论文或报告中写结果时，要注明使用的是这个固定 fold。
