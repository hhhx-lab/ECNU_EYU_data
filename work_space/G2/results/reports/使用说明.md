# reports 使用说明

这个目录只保留当前仍有用的轻量报告和实验模板，不再放旧沟通稿、重复 Word 文档或临时 smoke run 报告。

## 文件总览

| 文件 | 用途 |
|---|---|
| `G2_progress_report.md` | 当前 G2 结果区总索引，说明现有文件、清理内容和下一步。 |
| `g2_pretraining_execution_summary.md` | 基础审计脚本执行摘要，记录真实数据 manifest、统计、模板和脚本入口。 |
| `real_data_qc_summary.md` | 真实训练数据 QC 摘要，说明 final pass/fail、corrected label 与非法标签。 |
| `local_data_paths_check.md` | 本机外部数据路径检查，换训练机后需要重新核对。 |
| `ablation_plan_template.md` | real-only vs real+synth 消融计划模板，指标已对齐 2026 Task1 leaderboard。 |
| `README.md` | reports 目录极简说明。 |

## 质量报告模板在哪里

正式 G1 synthetic 批次的质量报告模板统一放在：

```text
work_space/G2/results/qc/G2_synthetic_data_QC报告模板_v2.md
```

这样 QC 规则、QC 表头、人工复查表和批次报告模板集中在 `results/qc/`，避免 reports 目录再出现重复模板。
