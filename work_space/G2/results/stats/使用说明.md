# stats 使用说明

这个目录保存真实训练数据的 label 和 lesion 分布统计，以及第一轮 synthetic data 的目标分布建议。它的核心作用是回答：真实数据中哪些 label、病灶大小、多发情况比较缺，G1 生成数据时应该优先补什么。

## 文件总览

| 文件 | 行数或内容 | 作用 |
|---|---:|---|
| `real_label_distribution.csv` | 1295 行 | 每个 final QC 通过病例的 label 体素数、体积和 label 组合 |
| `real_lesion_distribution.csv` | 9793 行 | 每个 lesion component 的体积、bbox、中心点、分档和 label 组成 |
| `real_lesion_distribution_summary.json` | 机器可读 summary | 给脚本或后续分析直接读取的统计摘要 |
| `real_lesion_distribution_summary.md` | 人可读 summary | 给人快速查看真实 lesion 分布 |
| `target_synthetic_distribution_v1.md` | 规则文档 | 第一轮 synthetic data 生成目标和限制 |

## label 含义

| label | 名称 | 含义 |
|---:|---|---|
| 0 | background | 背景 |
| 1 | NETC | non-enhancing tumor core |
| 2 | SNFH | surrounding non-enhancing FLAIR hyperintensity |
| 3 | ET | enhancing tumor |
| 4 | RC | resection cavity |

## `real_label_distribution.csv`

这个表是一例一行。它统计每个病例里 label 1-4 的体素数、体积、是否出现，以及 label 组合。

关键字段：

| 字段 | 含义 |
|---|---|
| `case_id` | 病例 ID |
| `label_1_voxels` 到 `label_4_voxels` | 各 label 的体素数 |
| `label_1_volume_mm3` 到 `label_4_volume_mm3` | 各 label 的体积，单位 mm3 |
| `has_NETC`, `has_SNFH`, `has_ET`, `has_RC` | 病例是否包含对应 label |
| `label_combination` | 该病例的 label 组合，例如 `NETC+SNFH+ET` |

这个表用于病例级分层。比如你想知道 RC 病例有多少、只含 ET 的病例有多少、synthetic data 应该不要过度生成哪类 label 组合，就看它。

## `real_lesion_distribution.csv`

这个表是一处 lesion component 一行。它不是病例级，而是病灶级统计。一个病例可能有很多行。

关键字段：

| 字段 | 含义 |
|---|---|
| `case_id` | 病例 ID |
| `lesion_id` | 该病例内的 lesion 编号 |
| `component_labels` | 该 lesion component 中包含的 label |
| `component_voxels` | component 体素数 |
| `component_volume_mm3` | component 体积 |
| `volume_bucket` | 体积分档：tiny、small、large |
| `bbox_i_min` 到 `bbox_k_max` | component 的 3D bbox |
| `center_i`, `center_j`, `center_k` | component 中心 |
| `has_ET`, `has_NETC`, `has_SNFH`, `has_RC` | component 内是否包含对应 label |

体积分档规则：

| 分档 | 含义 |
|---|---|
| `tiny_lt_27mm3` | 小于 27 mm3 的 tiny lesion |
| `small_27_to_275mm3` | 27 到 275 mm3 的 small lesion |
| `large_gt_275mm3` | 大于 275 mm3 的 large lesion |

这个表主要给 G1/G2 用来做生成策略：优先补 tiny/small、多发病例、稀有 label 组合，但也要避免 tiny lesion 生成过量。

## `real_lesion_distribution_summary.json`

这是机器可读摘要。脚本可以直接读取里面的计数，不需要每次重新扫 9793 行 lesion 表。

目前关键内容包括：

| 字段 | 含义 |
|---|---|
| `final_qc_pass_cases` | 通过最终 QC 的真实训练病例数，目前为 1295 |
| `cases_with_any_label` | 至少包含一个非背景 label 的病例数 |
| `label_combo_counter` | 各 label 组合的病例数 |
| `lesion_bucket_counter` | tiny/small/large lesion 数量 |
| `lesions_per_case` | 每例 lesion 数分布 |
| `cases_with_tiny_lesion` | 含 tiny lesion 的病例数 |

它适合给后续自动生成实验配置、抽样策略或图表脚本使用。

## `real_lesion_distribution_summary.md`

这是给人读的统计摘要。内容来自 summary JSON，但排版成 Markdown 表格和要点。

适合在组会、G1/G2 对接、写报告时直接打开看。它不作为程序输入，程序输入优先使用 JSON 和 CSV。

## `target_synthetic_distribution_v1.md`

这是第一轮 synthetic data 生成目标。它把真实分布转成生成约束，告诉 G1 第一阶段应该怎样保守生成。

核心原则：

1. 先交付 10-20 个 smoke cases，G2 先做 QC 和 nnU-Net 转换验证。
2. smoke 通过后再生成 100-300 个候选 synthetic cases。
3. 第一轮 accepted synthetic 不超过真实训练病例数的 25%。
4. source case 只来自 `final_qc_pass=true` 的训练病例，绝不来自 validation。
5. 优先补 small/tiny lesion 和多发病例，但 tiny lesion 比例不能失控。
6. RC 只基于真实 RC case 做保守变体，第一轮不凭空生成 RC。

后续如果真实训练消融显示某类 synthetic 有用，再迭代成 `v2`。
