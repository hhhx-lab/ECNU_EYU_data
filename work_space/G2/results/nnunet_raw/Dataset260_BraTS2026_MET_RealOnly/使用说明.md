# Dataset260_BraTS2026_MET_RealOnly 使用说明

这是 nnU-Net real-only baseline 的 raw dataset 占位目录。当前只有 `dataset.json` 和空的 `imagesTr/labelsTr` 目录，没有实际 NIfTI。

## 文件和目录

| 文件或目录 | 当前状态 | 用途 |
|---|---|---|
| `dataset.json` | 已生成 | 定义 nnU-Net 通道、label 名称和文件后缀 |
| `imagesTr/` | 当前只放说明文件 | 训练机上放四模态图像 |
| `labelsTr/` | 当前只放说明文件 | 训练机上放 label |
| `README.md` | 简短说明 | 当前目录的极简说明 |
| `使用说明.md` | 本文件 | 详细解释怎样使用这个占位目录 |

## `dataset.json`

当前内容表示：

| 字段 | 含义 |
|---|---|
| `channel_names.0` | t1n |
| `channel_names.1` | t1c |
| `channel_names.2` | t2w |
| `channel_names.3` | t2f |
| `labels.background` | 0 |
| `labels.NETC` | 1 |
| `labels.SNFH` | 2 |
| `labels.ET` | 3 |
| `labels.RC` | 4 |
| `numTraining` | 当前为 0，因为本机没有物化 NIfTI |
| `file_ending` | `.nii.gz` |
| `note` | 提醒使用 mapping 表在训练机上创建数据 |

训练机上真正物化完 1295 个 real-only 训练病例后，如果所用 nnU-Net 工具链要求 `numTraining` 与实际病例数一致，应把它改成 1295。

## 训练机上应该怎样物化

用 `work_space/G2/results/manifests/nnunet_case_mapping_realonly.csv` 作为唯一映射来源。对每一行执行：

| 源字段 | 目标文件名 |
|---|---|
| `t1n_source_path` | `imagesTr/{nnunet_case_id}_0000.nii.gz` |
| `t1c_source_path` | `imagesTr/{nnunet_case_id}_0001.nii.gz` |
| `t2w_source_path` | `imagesTr/{nnunet_case_id}_0002.nii.gz` |
| `t2f_source_path` | `imagesTr/{nnunet_case_id}_0003.nii.gz` |
| `seg_source_path` | `labelsTr/{nnunet_case_id}.nii.gz` |

例子：

| 类型 | 文件名 |
|---|---|
| t1n | `imagesTr/BraTSMET_000001_0000.nii.gz` |
| t1c | `imagesTr/BraTSMET_000001_0001.nii.gz` |
| t2w | `imagesTr/BraTSMET_000001_0002.nii.gz` |
| t2f | `imagesTr/BraTSMET_000001_0003.nii.gz` |
| seg | `labelsTr/BraTSMET_000001.nii.gz` |

## 注意事项

1. `imagesTr/` 和 `labelsTr/` 里不要提交真实 NIfTI 到 Git。
2. 本仓库当前在 `imagesTr/` 和 `labelsTr/` 内放了说明文件，只用于解释占位目录；正式训练目录如果要求纯 NIfTI，物化数据时不要复制这些说明文件，或在运行 nnU-Net 前删除。
3. 如果采用软链接，训练机路径必须稳定，避免训练中断链。
4. synthetic data 不应该直接加进这个 Dataset260；应为 real+synth 单独建数据集 ID，方便消融对比。
