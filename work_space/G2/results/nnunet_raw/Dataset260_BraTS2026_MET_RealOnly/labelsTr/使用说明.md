# labelsTr 使用说明

这个目录是 nnU-Net raw dataset 里放训练 label 的地方。当前仓库里没有真实 segmentation NIfTI，只放这份说明，避免误以为 label 已经物化完成。

## 这里将来应该放什么

训练机器上，每个病例放 1 个 label 文件，命名格式如下：

`{nnunet_case_id}.nii.gz`

例如 `BraTSMET_000001` 应对应：

`BraTSMET_000001.nii.gz`

源路径来自 `work_space/G2/results/manifests/nnunet_case_mapping_realonly.csv` 的 `seg_source_path` 字段。这个字段已经考虑了 corrected label，因此不要直接回原始数据目录临时找 `seg` 替代。

## label 值域

| label | 名称 |
|---:|---|
| 0 | background |
| 1 | NETC |
| 2 | SNFH |
| 3 | ET |
| 4 | RC |

## 注意事项

1. 不要把真实 NIfTI 提交进 Git。
2. 正式运行 nnU-Net 前，如果训练目录要求只包含 `.nii.gz`，请不要把这份说明文件复制到训练目录，或先删除。
3. label 文件必须与 `imagesTr` 里的四模态 shape、spacing、affine 对齐。
4. synthetic label 不放进这个 real-only Dataset260，应单独建 real+synth dataset。
