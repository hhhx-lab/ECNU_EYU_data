# nnunet_raw 使用说明

这个目录是 nnU-Net raw dataset 的轻量占位区。当前仓库里不复制 NIfTI，所以这里不是可直接训练的数据集，而是告诉 S1/S2 后续应该怎样在训练机器上物化数据。

## 当前结构

| 路径 | 内容 | 说明 |
|---|---|---|
| `Dataset260_BraTS2026_MET_RealOnly/` | real-only 数据集占位目录 | 对应 nnU-Net dataset ID 260 |
| `Dataset260_BraTS2026_MET_RealOnly/dataset.json` | 通道、label、文件后缀说明 | 可作为训练机物化后的 dataset.json 基础 |
| `Dataset260_BraTS2026_MET_RealOnly/imagesTr/` | 空目录 | 训练机上放四模态图像 |
| `Dataset260_BraTS2026_MET_RealOnly/labelsTr/` | 空目录 | 训练机上放 label |

## 为什么这里不放 NIfTI

1. 全量 NIfTI 数据太大，不适合进 Git。
2. 这个仓库只保存可追溯的小型契约文件。
3. 真正训练时应在 GPU/训练机器上根据 `manifests/nnunet_case_mapping_realonly.csv` 创建软链接或复制文件。

## 后续谁用

S1/S2 在训练机器上使用。G2 提供映射表和数据契约，S1/S2 负责把实际 NIfTI 物化成 nnU-Net 能读取的目录结构。

## 和 synthetic data 的关系

当前这个目录只对应 real-only baseline。synthetic data 通过 QC 后，应单独建立 real+synth 数据集目录，例如新的 Dataset ID，而不是直接混进这个 real-only 占位目录。
