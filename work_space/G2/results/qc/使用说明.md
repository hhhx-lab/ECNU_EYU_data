# qc 使用说明

这个目录定义 G2 如何接收、标准化、筛选和评价 G1 生成的 synthetic data。它不是训练代码，而是 G1 raw output 进入 nnU-Net 训练前的质量闸门，以及训练后判断 synthetic data 是否真的提升官方指标的记录入口。

## 文件总览

| 文件 | 内容 | 用途 |
|---|---|---|
| `G2_synthetic_data_QC规则策略_v2.md` | 完整 QC 策略 | G2 做 synthetic data 数据报告和质量检查的主标准 |
| `G2_official_metrics_alignment_QC_strategy_2026-06-15.md` | 官方指标对齐策略 | 明确官方 leaderboard 字段、G2 当前 QC 是否符合、训练后如何验收 |
| `official_leaderboard_metrics_template.csv` | 官方字段 CSV 模板 | 记录 real-only 与 real+synth 的官方同款验证结果 |
| `qc_metrics_template_v2.csv` | v2 自动 QC 指标模板 | 逐例记录 G1 run、source、几何、标签、ROI、lesion、teacher 和最终结论 |
| `diffusion_quality_metrics_template.csv` | 扩散生成质量专项模板 | 衡量 ROI 边界、z 连续性、多模态一致性、复制风险、医学 feature FID/MMD |
| `qc_case_review_template.csv` | 人工复查模板 | 记录看图复查原因、结果和 reviewer |
| `G2_synthetic_data_QC报告模板_v2.md` | 数据报告模板 | 每批 synthetic data 完成 QC 后复制填写 |
| `UCSD_T2W_内容异常检查报告_2026-06-14.md` | 真实数据 T2W 核查报告 | 记录 T2W 坏文件检查和 gzip header fake T2W 识别 |
| `official_t2w_gzip_header_audit_2026-06-15.csv` | 全量 T2W gzip header audit | 每个训练病例一行，记录 gzip 原始文件名和 fake 判定 |
| `official_fake_t2w_cases_by_gzip_header_2026-06-15.csv` | fake T2W 清单 | 以 gzip header 原始文件名识别官方 fake/synthetic T2W |
| `official_non000_t2w_cases_2026-06-15.csv` | 非 000 编号清单 | 辅助追踪病例后缀，不能作为 fake T2W 最终判据 |

旧的 `qc_rules_v1.md`、`qc_metrics_template.csv` 和 2026-06-14 smoke run 输出已经清理，正式批次统一使用 v2 文件。

## 推荐检查顺序

1. 先读 `G2_official_metrics_alignment_QC_strategy_2026-06-15.md`，明确官方榜单字段和 G2 的验收边界。
2. 再读 `G2_synthetic_data_QC规则策略_v2.md`，执行 L0-L12 分层 QC。
3. 拿到 G1 输出后，先做 raw intake：解析 raw case ID、legacy suffix、source、label_kind、label_index。
4. 如果 G1 没有 manifest，先用 `../manifests/synthetic_generation_manifest_template_g1.csv` 补建。
5. 按 `qc_metrics_template_v2.csv` 逐例生成自动 QC 指标。
6. 按 `diffusion_quality_metrics_template.csv` 生成扩散专项质量指标。
7. 对触发人工复查的病例，填写 `qc_case_review_template.csv`。
8. 按 `G2_synthetic_data_QC报告模板_v2.md` 形成批次数据报告。
9. 只有报告结论为 `accepted_for_training` 或 `accepted_for_ablation_only` 的样本，才允许进入 real+synth 消融。
10. S1/S2 训练出预测后，把 real-only 与 real+synth 结果写入 `official_leaderboard_metrics_template.csv` 同款字段。

推荐脚本入口：

1. `../../code/g2_synthetic_raw_intake_qc.py`：接收 G1 raw run 并输出 QC 闭环。
2. `../../code/g2_materialize_nnunet_dataset.py`：把 accepted 数据转成训练机 nnU-Net raw 入口。
3. `../../code/g2_official_mets_metrics_parser.py`：解析或校验官方同款指标。

## 当前官方指标结论

你贴出的排行榜字段与官方 Task1 Results leaderboard 当前字段一致。G2 的现有 QC 主要检查“数据能不能安全进入训练”，不是官方榜单指标本身。真正证明 synthetic data 有用，必须看训练后的官方同款指标：

1. ET/RC/TC/WT 的 lesionwise DSC。
2. ET/RC/TC/WT 的 lesionwise NSD。
3. ET/TC/WT/RC 的 small-instance TP/FN/FP/F1。

`AUC over F1` 和 `HD95` 可以保留作内部辅助分析，但不能作为当前主报告的官方指标。

## 当前 G2 QC 是否专业有效

当前 G2 QC 是有效的，因为它覆盖了 synthetic data 最容易出问题的关键环节：

1. 文件完整性：四模态和 seg 必须齐全。
2. NIfTI 可读性：不能有坏文件、NaN/Inf、常数图。
3. 几何一致性：病例内 shape、spacing、affine、orientation 必须一致。
4. 标签合法性：seg 只能含 `{0,1,2,3,4}`。
5. source 合规性：不能来自 fixed validation fold 或 official validation。
6. ROI 插入合理性：不能改动非 ROI、不能把 lesion 放到脑外。
7. lesion 结构：记录 tiny `<27 mm3`、small `27-275 mm3`、large `>275 mm3`。
8. 多模态医学一致性：ET 看 t1c，SNFH 看 t2f/t2w，RC 单独复核。
9. 扩散伪影和复制风险：看边界、z 连续性、source-synth 相似性和同批重复。
10. 人工复查：所有临界病例必须有 reviewer 记录。

它还需要配合 official-metric ablation 才能回答“是否提高比赛成绩”。

## 批次放行标准

### 进入训练前

1. `validation_leakage_count=0`
2. `label_values_valid_rate=100%`
3. `geometry_consistent_rate=100%`
4. `nifti_readable_rate=100%`
5. `hard_reject_rate<=5%`
6. `manual_review_unresolved_count=0`
7. `nnunet_integrity_pass=true`

### 训练后

1. `lesionwise_dsc_mean_et/rc/tc/wt` 不下降。
2. `lesionwise_nsd_mean_et/rc/tc/wt` 不下降。
3. `small_instance_f1_et/tc/wt/rc` 不下降。
4. `small_instance_fn_*` 不增加。
5. `small_instance_fp_*` 不明显增加；如果 FP 增加超过 10% 且 TP 没有增加，不进入主训练。

如果只通过进入训练前 QC，只能写“QC 质量优秀”；只有 official-metric ablation 也通过，才能写“训练价值优秀”。
