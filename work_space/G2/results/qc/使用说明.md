# qc 使用说明

这个目录定义 G2 如何接收和筛选 G1 生成的 synthetic data。它不是训练代码，而是 synthetic data 进入训练前的质量闸门。

## 文件总览

| 文件 | 内容 | 用途 |
|---|---|---|
| `G2_synthetic_data_QC规则策略_v2.md` | 完整 QC 策略 | G2 做 synthetic data 数据报告和质量检查的主标准 |
| `qc_rules_v1.md` | 已扩写为 v2 执行摘要 | 快速查看强制拒绝、人工复查、批次放行标准 |
| `qc_metrics_template.csv` | 表头模板，当前 0 行数据 | G1 输出 synthetic cases 后，G2 按这个模板逐例填 QC 指标 |
| `qc_metrics_template_v2.csv` | v2 自动 QC 指标模板 | 推荐后续正式批次使用，覆盖 source、几何、标签、lesion、teacher 和最终结论 |
| `qc_case_review_template.csv` | 人工复查模板 | 记录看图复查原因、结果和 reviewer |
| `G2_synthetic_data_QC报告模板_v2.md` | 数据报告模板 | 每批 synthetic data 完成 QC 后复制填写 |

## 推荐检查顺序

1. 先读 `G2_synthetic_data_QC规则策略_v2.md`，明确完整 QC 分层、检查方法和放行标准。
2. 拿到 G1 输出后，先检查 run manifest、generation log 和每例 metadata。
3. 按 `qc_metrics_template.csv` 逐例生成自动 QC 指标。
4. 对触发人工复查的病例，填写 `qc_case_review_template.csv`。
5. 按 `G2_synthetic_data_QC报告模板_v2.md` 形成批次数据报告。
6. 只有报告结论为 `accepted_for_training` 或 `accepted_for_ablation_only` 的样本，才允许进入 real+synth 消融。

## `G2_synthetic_data_QC规则策略_v2.md`

这是当前最完整的规则文件。它把 QC 拆成 L0-L10：

| 层级 | 内容 |
|---|---|
| L0 | 交付完整性与可追溯性 |
| L1 | 文件与命名 |
| L2 | NIfTI header、shape、spacing、affine |
| L3 | 图像数组和 label 值域 |
| L4 | source 与 validation leakage |
| L5 | lesion-level 结构和体积分布 |
| L6 | 图像生成痕迹、ROI 边界、z 轴连续性 |
| L7 | teacher model 辅助检查 |
| L8 | 批次分布 |
| L9 | nnU-Net integrity check |
| L10 | real-only vs real+synth 消融放行 |

这份文件也写明了 2026 Task1 的任务依据、当前真实数据参考分布、往年数据校验经验，以及数据报告必须回答的问题。

## `qc_rules_v1.md`

这个文件写的是判定标准。G1 每交付一批 synthetic data，G2 应先按这里过一遍，而不是直接混入训练。

规则分三类：

| 类别 | 含义 |
|---|---|
| 强制拒绝 | 出现后这个 synthetic case 不能进训练 |
| 需要人工复查 | 不能自动通过，需要人工看图或看统计 |
| 只记录不拒绝 | 作为质量指标记录下来，但不单独作为拒绝条件 |

强制拒绝的典型情况包括：

1. 缺少任一模态或缺少 `seg.nii.gz`。
2. 四模态与 seg 的 shape、spacing 或 affine 不一致。
3. 图像含 NaN/Inf，label 非整数。
4. label 值域不在 `{0,1,2,3,4}`。
5. synthetic lesion 出现在脑外全零背景区。
6. ROI 边界出现明显方块断层。
7. z 轴严重断裂。
8. 缺少生成 manifest 或日志，无法追溯来源。

这个文件现在保留原文件名，但内容已扩写为 v2 执行摘要。后续每一批 synthetic data 必须注明使用的 QC 规则版本。

## `qc_metrics_template.csv`

这是 synthetic case 的 QC 记录表模板，目前只有表头，没有数据行。G1 交付第一批 synthetic cases 后，复制这个表头生成正式 QC 表，例如 `qc_metrics_g1_smoke_v1.csv`。

字段可以按用途分成几组：

| 字段组 | 字段 | 含义 |
|---|---|---|
| 身份与来源 | `case_id`, `source_case_id`, `generation_run_id`, `generator_checkpoint`, `seed`, `generation_mode` | 说明 synthetic case 从哪个真实病例、哪次生成、哪个 checkpoint 和 seed 来 |
| 文件完整性 | `has_all_modalities`, `has_seg` | 检查四模态和 seg 是否齐全 |
| 空间一致性 | `shape_consistent`, `spacing_consistent`, `affine_valid` | 检查图像和 label 是否能一起训练 |
| 数值合法性 | `has_nan_or_inf`, `label_values_valid`, `label_is_integer`, `empty_mask` | 检查图像数值和 label 值域 |
| lesion 统计 | `lesion_count`, `min_lesion_volume_mm3`, `max_lesion_volume_mm3`, `tiny_lesion_count`, `small_lesion_count`, `large_lesion_count` | 记录病灶数量和大小分布 |
| 图像质量 | `roi_boundary_score`, `z_continuity_score` | 记录 ROI 边界和 z 轴连续性质量 |
| teacher 参考 | `teacher_dice`, `teacher_lesion_count_diff` | 用已有分割模型粗看 synthetic label 是否离谱 |
| 最终决定 | `qc_pass`, `qc_reject_reason`, `manual_review_required` | 是否通过、拒绝原因、是否需要人工复查 |

使用时建议：

1. 一行对应一个 synthetic case。
2. 不要删除字段，暂时没有的指标留空。
3. `qc_pass=False` 时必须写 `qc_reject_reason`。
4. `manual_review_required=True` 的样本不能直接进训练，必须人工确认。
5. 最终进入训练的数据，应另存 accepted manifest，而不是直接从 QC 表里临时筛。

正式批次建议优先使用 `qc_metrics_template_v2.csv`。旧模板保留是为了兼容前一轮文档；v2 模板字段更完整，能直接支撑数据报告。

## `qc_metrics_template_v2.csv`

这是正式推荐的自动 QC 指标表。字段分组如下：

| 字段组 | 用途 |
|---|---|
| 身份与来源 | `case_id/source_case_id/run_id/checkpoint/seed` |
| 交付完整性 | metadata、manifest、log、五个 NIfTI 是否存在 |
| 几何一致性 | shape、spacing、affine、orientation |
| 图像数值 | NaN/Inf、常数图、每模态强度分位数 |
| label 合法性 | 整数性、值域、空 mask |
| source 与泄漏 | 是否来自 final QC pass train、是否固定 val fold、是否 official validation |
| lesion 结构 | lesion 数、tiny/small/large、体积、label 组合、bbox |
| 图像质量 | ROI 边界、z 连续性、强度漂移、伪影 |
| teacher 辅助 | teacher Dice 和 lesion count diff |
| 最终结论 | hard reject、manual review、qc decision、是否 accepted |

## `qc_case_review_template.csv`

这是人工看图复查表。触发 manual review 的病例必须逐例填写，不允许只在报告里写“已看过”。

重点字段：

| 字段 | 含义 |
|---|---|
| `review_priority` | 复查优先级 |
| `review_reason` | 触发复查的原因 |
| `viewed_t1c_et` | 是否查看 t1c + ET overlay |
| `viewed_t2f_snfh` | 是否查看 t2f + SNFH overlay |
| `viewed_three_planes` | 是否查看三方向 |
| `roi_boundary_ok` | ROI 边界是否自然 |
| `z_continuity_ok` | z 轴是否连续 |
| `label_anatomy_ok` | label 空间关系是否合理 |
| `review_decision` | `accept/reject/ablation_only/regenerate` |

## `G2_synthetic_data_QC报告模板_v2.md`

这是每批 synthetic data 的正式数据报告模板。报告结论只能写四类：

1. `accepted_for_training`
2. `accepted_for_ablation_only`
3. `needs_regeneration`
4. `rejected`

报告必须回答：生成了多少、通过多少、拒绝原因、是否有 validation leakage、label 是否合法、几何是否一致、lesion 分布是否合理、是否通过 nnU-Net integrity check，以及是否允许进入 real+synth 消融。
