# Training `t2w` 内容异常与 fake 标记检查报告

- 检查日期：2026-06-14
- 补充修订日期：2026-06-15
- 检查对象：`MICCAI-LH-BraTS2025-MET-Challenge-Training`
- 重点补充对象：`MICCAI-LH-BraTS2025-MET-Challenge-Training/UCSD - Training`
- 检查目的：核实真实训练集里的 `t2w` 是否存在空文件、坏文件、内容层局部异常、边缘裁切、以及与其他模态的局部不一致；同时补充识别官方 `t2w.nii.gz` gzip header 内部原始文件名中带 `fake` 的样本。

## 1. 检查范围

本次只看真实数据集，不看 G2 清单推断结果，直接回到原始病例目录和官方 zip 内部路径逐项核查。2026-06-15 修订版已覆盖整个训练集，包括根目录病例和 `UCSD - Training` 子目录病例。

覆盖内容：

1. 每个病例目录是否齐全。
2. 每个病例的 `t1n / t1c / t2w / t2f / seg` 是否都存在。
3. `t2w` 文件是否可正常打开、是否空文件、是否损坏。
4. `t2w` 是否存在切片级、体素级内容异常。
5. `t2w` 与 `t1n / t1c / t2f` 的局部表现是否一致。
6. 对可疑病例做 3D 三视图人工抽样复核。
7. 读取 `t2w.nii.gz` 的 gzip header 原始文件名，查找内部原始文件名是否包含 `fake`。

## 2. 检查方法

### 2.1 文件级完整性

对每个病例检查：

- `*-t2w.nii.gz` 是否存在。
- 文件大小是否为 0。
- gzip 头是否正常。
- NIfTI 是否可读。
- 数据是否能完整读出。
- 数据是否为全常数、是否含 NaN/Inf。

### 2.2 内容级异常

对 `t2w` 做切片和体素级筛查，重点看：

- 是否有中间空层。
- 是否有连续断层。
- 是否有边缘被裁掉的迹象。
- 是否有局部坏块或大面积空洞。
- 切片间强度是否出现异常跳变。

### 2.3 多模态对照

对同病例的 `t1n / t1c / t2w / t2f` 进行并列检查，重点看：

- 局部结构是否一致。
- 是否出现某一模态单独缺块。
- 是否存在明显错位或内容异常。

### 2.4 3D 三视图人工复核

对筛出的候选病例生成轴位、冠状位、矢状位三方向拼图，做人工目视判断。

### 2.5 gzip header 内部原始文件名识别

官方任务说明中已明确：2025 年后 T2W 在 BraTS-METS 中变为非强制序列，部分病例含原生 T2，部分病例含合成 T2，部分病例没有 T2 序列。实际数据包中本批训练数据均有 `t2w` 文件，但部分 `t2w.nii.gz` 在 gzip header 中保留了原始文件名，例如：

```text
BraTS-MET-00577-002-t2w.nii.gz
  gzip original filename = 100132C_T1post_fake_B.nii
```

因此，本报告修正判据为：

- **不能仅凭病例编号后缀 `001/002/...` 判断 fake。**
- **应以 `t2w.nii.gz` 的 gzip header 原始文件名是否包含 `fake` 作为 fake T2W 的直接判据。**
- 病例编号后缀只能作为辅助信息，不能作为最终 fake 判定标准。

## 4. 结论

本次对训练集 `t2w` 做了从文件级到内容级的多层检查，结论是：

1. **没有发现空文件或坏掉的 `t2w`。**
2. **没有发现明确的局部坏块、边缘裁切或中间断层。**
3. **没有发现与 `t1n / t1c / t2f` 的显著局部不一致。**
4. **训练集中存在 265 个 gzip header 原始文件名带 `fake` 的 `t2w`，这些应明确标注为 fake / synthetic T2W。**
5. **`UCSD - Training` 646 个 `t2w` 已纳入全量 audit，未发现 gzip header 原始文件名含 `fake`。**
6. **病例编号后缀 `001/002/...` 不能直接等价于 fake；例如 fake T2W 中也有 177 个后缀为 `000` 的病例。**
7. **当前未能确认任何需要按坏文件硬性剔除的 `t2w` 异常病例；但 gzip header 原始文件名含 `fake` 的病例应在后续 manifest、训练划分和报告中单独记录。**

## 5. 备注

- 前期出现的高风险候选，主要来源于切片强度变化和背景占比的启发式排序，不等于真实坏文件。
- 这次人工复核确认，候选病例整体更接近正常扫描差异。
- 本报告早期检查重点是“缺失 / 坏文件 / 内容损坏”，没有读取 `.nii.gz` 的 gzip header 原始文件名；2026-06-15 已补充修正。
- 如果后续做训练数据 manifest，必须新增字段记录 `case_suffix`、`gzip_original_filename`、`t2w_is_fake_by_gzip_header`、`t2w_official_status`，避免把所有 `t2w` 都当成同一种来源。
