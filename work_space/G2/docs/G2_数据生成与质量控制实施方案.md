更新日期：2026-06-13

对应分工：G2，数据生成、G1 raw output 接收、synthetic data QC、nnU-Net 导出与消融评估
当前主口径：G2 主动适配 G1，不要求 G1 第一阶段重构成 G2 理想资产格式

## 0. 最终结论

1. G1 现有模型路线可以作为第一阶段主线保留：它本质是沿用 GliGAN 工程习惯的 `96x96x96` 局部 ROI 生成/修复系统，而不是完整 3D 脑 MRI 从零生成器。
2. G2 不应要求 G1 第一版直接输出 `SYN-MET-* + metadata.json + nnU-Net ready` 的理想数据资产。这样会迫使 G1 大改他人的模型代码，风险高，收益低。
3. G2 应把自己定位成 G1 输出的 adapter、auditor 和 publisher：G1 只要稳定生成 raw synthetic case，G2 负责把它转成可追溯、可审计、可训练、可报告的数据资产。
4. 第一阶段主接口采用 `GliGAN-compatible raw output`：CSV 驱动、真实 source case、局部 ROI、四模态回填、legacy case 命名、G2 下游重命名。
5. G2 最重要的任务不是追求 synthetic 图像“看起来好”，而是证明每个 synthetic case 来源清楚、空间合法、标签合法、病灶合理、无验证/测试集泄漏、可被 nnU-Net 读取，并且在固定 internal test 上不伤害模型表现。
6. G2 的所有数据报告都必须能回答：这个 synthetic case 从哪里来、由哪个 checkpoint 和 seed 生成、插入了什么 label、改了哪个 ROI、为什么通过或拒绝、能不能进入训练。

## 1. G1 当前模型事实

G2 后续所有设计都以 G1 真实代码为准，而不是以理想契约想象为准。

### 1.1 G1 的实际生成单位

G1 的 diffusion 分支围绕以下对象工作：

```text
source case 的某个模态 crop
  + label crop
  + Gaussian noise / inpainting noise
  -> diffusion model
  -> reconstructed 96x96x96 tumour patch
  -> 回填到 source case
```

G1 不是直接生成完整病例的全脑扩散模型。它更接近 GliGAN 的 modality generator 替代版。

### 1.2 G1 的输入口径

G1 读取 GliGAN 风格 CSV，核心列为：

```csv
id,scan_t1ce,scan_t2,scan_flair,scan_t1,label,center_x,center_y,center_z,x_extreme_min,x_extreme_max,y_extreme_min,y_extreme_max,z_extreme_min,z_extreme_max,x_size,y_size,z_size
```

G2 必须继续给 G1 这个口径，同时额外保留 G2 自己的扩展列用于 QC、防泄漏和报告。

### 1.3 G1 的输出口径

G1 第一阶段允许输出 legacy raw case，例如：

```text
BraTS-MET-00007-000_fake_label_0/
  BraTS-MET-00007-000_fake_label_0-scan_t1.nii.gz
  BraTS-MET-00007-000_fake_label_0-scan_t1ce.nii.gz
  BraTS-MET-00007-000_fake_label_0-scan_t2.nii.gz
  BraTS-MET-00007-000_fake_label_0-scan_flair.nii.gz
  BraTS-MET-00007-000_fake_label_0-seg.nii.gz
```

或直接输出 2026 后缀：

```text
BraTS-MET-00007-000_fake_label_0/
  BraTS-MET-00007-000_fake_label_0-t1n.nii.gz
  BraTS-MET-00007-000_fake_label_0-t1c.nii.gz
  BraTS-MET-00007-000_fake_label_0-t2w.nii.gz
  BraTS-MET-00007-000_fake_label_0-t2f.nii.gz
  BraTS-MET-00007-000_fake_label_0-seg.nii.gz
```

G2 两种都接收。最终进入训练前由 G2 统一重命名和转换。

## 2. G2 新定位

G2 不再把自己写成“等待 G1 输出标准 synthetic asset 的下游使用者”。G2 是完整数据资产链的负责人。

### 2.1 G2 负责的事情

1. 生成 G1 可读的 source CSV。
2. 控制哪些真实病例允许作为 source。
3. 控制 internal val/test holdout 不被 G1 用作训练或 synthetic source。
4. 接收 G1 legacy raw output。
5. 解析 raw case 命名，恢复 source、label kind、label index。
6. 兼容 legacy 模态后缀并映射到 2026 后缀。
7. 补建 `synthetic_generation_manifest.csv`。
8. 补建 `generation_log.jsonl` 的审计视图，至少生成 G2 sidecar log。
9. 为每个 synthetic case 分配 `SYN-MET-*` 或 nnU-Net case ID。
10. 跑 L0-L12 QC。
11. 生成人工复查表和数据报告。
12. 只把 accepted cases 导出给 S1/S2。
13. 做 real-only vs real+synth 消融。

### 2.2 G1 第一阶段只需要保证的事情

1. 能基于 G2 source CSV 读取病例。
2. 不使用 G2 标记为 forbidden 的 source。
3. 输出每例四模态和 `seg`。
4. NIfTI 可以被 `nibabel` 读取。
5. 每例尽量继承 source case 的 affine/header。
6. 说明 checkpoint、seed、生成模式、label channel、noise type。
7. 若生成失败，要保留失败信息或控制台日志。

这就是 G1 第一阶段的最低要求。G2 不强制 G1 第一版输出 per-case `metadata.json`。

## 3. 三层数据契约

G2 采用三层契约，避免把 G1 raw output 和最终训练资产混在一起。

| 层级 | 名称 | 负责人 | 作用 |
|---|---|---|---|
| L1 | G1 raw 层 | G1 生成，G2 接收 | GliGAN-compatible raw synthetic cases |
| L2 | G2 memory 层 | G2 | manifest、log、QC、source 追踪、拒绝理由 |
| L3 | training export 层 | G2 | nnU-Net raw dataset、accepted manifest、ablation 版本 |

数据流固定为：

```text
real train data
  -> G2 real_train_manifest
  -> G2 g1_gligan_source_cases_v1.csv
  -> G1 raw synthetic output
  -> G2 raw intake
  -> G2 synthetic_generation_manifest
  -> G2 QC metrics + review
  -> accepted_synthetic_manifest
  -> nnU-Net real+synth dataset
  -> fixed internal test ablation
```

## 5. G2 写给 G1 的 source CSV

1. 来自 real train，不来自官方 validation。
2. 不在固定 internal val/test holdout。
3. corrected overlay 后 label 值域合法。
4. `final_qc_pass=True`。
5. 不属于明确排除病例。
6. 对于第一阶段 GliGAN-compatible ROI，目标 lesion bbox 任一轴不超过 96。

### 5.2 当前已知事实

### 5.3 G1 CSV 保留扩展列

## 6. G1 raw intake 规则

G2 收到 G1 输出后，第一步不是训练，而是 intake。

### 6.1 raw case 识别

从目录名解析：

```text
<source_case_id>_<label_kind>_label_<label_index>
```

字段含义：

| 字段 | 示例 | 含义 |
|---|---|---|
| `source_case_id` | `BraTS-MET-00007-000` | 被插入 synthetic lesion 的真实病例 |
| `label_kind` | `fake` / `real` | fake 表示生成 label，real 表示借用真实 label |
| `label_index` | `0` | 同一 source 下的第几个 synthetic sample |

如果目录名无法解析，G2 不自动拒绝，但必须标记 `manual_review_required=True`，并且不进入训练，除非人工补全 source 和 seed。

### 6.2 模态后缀映射

| G1 legacy 后缀 | G2/2026 后缀 | nnU-Net 通道 |
|---|---|---|
| `scan_t1` | `t1n` | `0000` |
| `scan_t1ce` | `t1c` | `0001` |
| `scan_t2` | `t2w` | `0002` |
| `scan_flair` | `t2f` | `0003` |
| `seg` | `seg` | labelsTr |

G2 最终导出时固定通道顺序：

```text
0000 = t1n
0001 = t1c
0002 = t2w
0003 = t2f
```

### 6.3 记忆化输出

每个 raw case 必须被 G2 转成一行 `synthetic_generation_manifest`。即使 G1 没有给 manifest，G2 也要补建。

最少字段：

```csv
synthetic_raw_id,synthetic_final_id,source_case_id,label_kind,label_index,generation_run_id,generator_name,generator_checkpoint,label_channels,rc_policy,noise_type,seed,output_suffix_scheme,source_allowed,source_in_fixed_val,source_from_official_validation,raw_case_dir,status,qc_status,accepted_for_training
```

推荐字段：

```csv
generator_checkpoint_t1n,generator_checkpoint_t1c,generator_checkpoint_t2w,generator_checkpoint_t2f,label_source_case_id,label_component_id,insert_center_x,insert_center_y,insert_center_z,roi_x_min,roi_x_max,roi_y_min,roi_y_max,roi_z_min,roi_z_max,source_shape_x,source_shape_y,source_shape_z,output_shape_x,output_shape_y,output_shape_z,error_type,error_message
```

## 8. QC 总策略

G2 的 QC 不是一个分数，而是一套闸门。

### 8.1 先硬拒绝，再评分

以下情况无论图像看起来多好，都不能进入训练：

1. source 来自固定 internal val/test holdout。
2. source 来自官方 validation。
3. source 缺失，无法追溯。
4. 缺任一模态或缺 `seg`。
5. NIfTI 无法读取。
6. 同一 case 内 shape/spacing/affine 不一致。
7. 图像有 NaN/Inf 或常数图。
8. label 非整数或值域不在 `{0,1,2,3,4}`。
9. lesion 大面积在脑外。
10. ROI 边界出现严重方块断层。
11. z 轴出现严重切片断裂。
12. 没有 manifest/log，且 G2 无法补建关键追踪字段。

### 8.2 通过硬检查后再分级

| 等级 | 名称 | 使用方式 |
|---|---|---|
| A | excellent | 可进入训练，并作为优质展示样例 |
| B | accepted | 可进入 real+synth 训练 |
| C | ablation_only | 只进入受控消融，不进入主训练集 |
| D | needs_regeneration | 返回 G1 重生成或调整参数 |
| F | rejected | 不可训练，不可消融 |

分级依据：

1. ROI 边界自然程度。
2. z 轴连续性。
3. 多模态一致性。
4. label 解剖合理性。
5. lesion 体积分布。
6. source 使用次数和分布偏差。
7. teacher model 辅助一致性。
8. 人工复查结论。

## 9. 如何判断数据质量非常优秀

一个 synthetic batch 只有同时满足以下条件，才能称为“质量非常优秀”。

### 9.1 单例优秀

1. hard reject 项全部为 false。
2. source、checkpoint、seed、label_kind、raw ID、final ID 全部可追溯。
3. 五个 NIfTI 全部可读，病例内 shape/spacing/affine 一致。
4. `seg` 值域只含 `{0,1,2,3,4}`。
5. 插入 lesion 在脑内，不覆盖 source 原有 lesion。
6. ROI 边界没有明显立方体接缝。
7. 相邻 slice 的 lesion 面积和强度变化连续。
8. t1c 上 ET 表现合理，t2f/t2w 上 SNFH 表现合理。
9. label 组合不违反 2026 MET 语义。
10. teacher model 不出现完全相反的异常提示，或异常已人工确认。

### 9.2 批次优秀

1. validation leakage 为 0。
2. hard reject rate 不高于 5%。
3. unresolved manual review 为 0。
4. accepted synthetic 数量第一轮不超过真实训练病例 25%。
5. 每个 source case 默认最多贡献 1 个 accepted synthetic。
6. tiny/small/large 分布与真实训练集接近，不人为制造大量噪点 tiny lesion。
7. RC synthetic 只来自真实 RC 语境，不凭空生成大量 RC。
8. source case 分布不集中在少数病例。
9. 通过 `nnUNetv2_plan_and_preprocess --verify_dataset_integrity`。
10. real+synth 在固定 internal test 上不降低核心指标。

### 9.3 训练有用

最终采用标准不是“好看”，而是：

1. real+synth 相比 real-only，Dice/NSD 不下降。
2. 官方 small-instance TP/FN/FP/F1 不下降，尤其 `small_instance_f1_et/tc/wt/rc` 不能被 synthetic 拉坏。
3. tiny lesion recall 有提升或不下降。
4. large lesion segmentation 不被牺牲。
5. RC 相关指标不被 synthetic data 拉坏。
6. 如果只在某类病灶提升，报告必须写清楚适用范围。

## 10. G2 每批执行流程

### 10.1 G1 smoke run

1. G2 给 G1 提供 `g1_gligan_source_cases_v1.csv`。
2. G1 生成 10-20 个 raw cases。
3. G2 执行 intake。
4. G2 补建 manifest/log。
5. G2 跑 L0-L9 QC。
6. G2 抽样人工看图。
7. G2 导出 smoke nnU-Net dataset。
8. G2 给 G1 回一份 reject reason 表。

smoke 通过标准：

1. 10-20 cases 可读。
2. 自动 QC 通过率不低于 80%。
3. validation leakage 为 0。
4. 人工抽查无系统性方块伪影。
5. 能无手工改名导出 nnU-Net 测试数据集。

### 10.2 G1 v1 batch

1. 扩大到 100-300 个候选 synthetic。
2. 每个 source 默认最多 1 个 accepted。
3. 对所有 rejected case 写明原因。
4. 对 needs_regeneration case 回传 G1。
5. accepted case 才导出 Dataset261/262。
6. S1/S2 做 real-only vs real+synth 消融。

### 10.3 回滚规则

以下任一情况出现，整批或相关子批次回滚：

1. 发现 validation leakage。
2. 发现 label 值域污染。
3. 发现 source 追踪缺失且无法恢复。
4. accepted batch 训练后明显降低真实验证指标。
5. 某个 checkpoint 生成的 case 集中出现同类伪影。
6. RC synthetic 明显损害 RC 类表现。
