## 0. 调整后的核心结论

1. G2 的方向不需要推翻：用 diffusion model 替代往年 GAN 做 synthetic data augmentation 是合理的。
2. 但第一阶段不能把目标设成“从零生成完整 3D 脑 MRI 病例”。更稳妥的第一版路线应是：保留真实病例脑背景，只在局部 ROI 内生成或替换病灶区域。
3. 往年 GliGAN 的可复用思想不是“GAN 本身”，而是“真实背景 + 合理 label/ROI + 局部生成 + 可追踪数据工程 + 下游消融验证”。
4. 因此，今年 G2 应把 diffusion 理解为 GliGAN 中 modality generator 的升级替代，而不是把整个数据系统重写成一个不可控的全病例生成器。
5. G2 当前最重要的工作不是等待 G1 模型训练完成，而是先把真实数据、数据契约、manifest、QC、nnU-Net 转换、固定验证划分和消融框架准备好。
6. 最终判断 synthetic data 是否有用，不看“生成图像是否好看”，也不只看 FID/MS-SSIM，而要看固定真实验证集上的 Dice、NSD、病灶检测 F1/AUC、小病灶召回和假阳性是否改善。

## 1. G2 在团队中的准确定位

### 1.1 G2 不是单纯的数据生成执行者

G2 的任务不是简单运行 G1 模型生成更多病例。G2 的核心职责是把生成模型输出变成分割组可以安全使用、可以复现、可以审计、可以验证的数据资产。

G2 需要负责以下闭环：

1. 真实数据清点。
2. corrected labels 覆盖。
3. 真实训练集 label 和 lesion 分布统计。
4. 和 G1 定义生成输入输出契约。
5. 接收或调度 G1 diffusion 生成 synthetic raw cases。
6. 建立 synthetic manifest。
7. 做基础 QC、label QC、图像 QC、多模态一致性 QC、3D 连续性 QC。
8. 转换到 nnU-Net v2 的 `imagesTr + labelsTr` 格式。
9. 保证 validation 只含真实病例，避免同源 synthetic 泄漏。
10. 和 S1/S2 做真实验证集消融。
11. 输出质量报告，指导下一轮 G1/G2 迭代。

### 1.2 G2 的第一性问题

G2 所有工作都应围绕一个问题展开：

```text
这批合成数据进入训练后，是否能提升真实验证集上的脑转移瘤分割和检测表现？
```

如果一批 synthetic data 满足图像视觉真实、FID 好看、MS-SSIM 合理，但加入训练后造成小病灶假阳性上升、检测 F1 下降，那么这批数据对比赛目标就是失败的。

### 1.3 G2 不应做的事

1. 不应只生成 PNG 或截图展示，必须交付可训练的 NIfTI 数据。
2. 不应把 synthetic data 直接混进真实训练集而不记录来源。
3. 不应使用 validation 病例派生 synthetic training data。
4. 不应静默修复非法标签值，所有修复必须记录规则和病例 ID。
5. 不应照搬成人胶质瘤的标签 mapping、连通域阈值、后处理规则。
6. 不应在第一阶段追求海量生成。第一阶段目标是跑通闭环。
7. 不应把 G1 输出格式交给 G1 自由发挥，必须提前锁定契约。

## 2. Task 1 对 G2 的关键约束

### 2.1 官方任务目标

BraTS 2026 Task 1 是脑转移瘤分割与检测任务。算法需要覆盖治疗前和治疗后病例，自动检测并分割不同大小的脑转移瘤、周围水肿和切除腔。

### 2.2 标签体系

| 标签 | 缩写 | 名称 | G2 生成约束 |
|---:|---|---|---|
| 0 | background | 背景 | 病灶不能生成到脑外、零背景区或明显非脑组织区域。 |
| 1 | NETC | 非强化肿瘤核心 | 通常不应孤立漂浮，应与 ET 或肿瘤核心结构存在合理关系。 |
| 2 | SNFH | 周围非强化 FLAIR 高信号 | 应在 FLAIR/T2W 上表现为合理高信号，不能边界过硬或远离病灶主体。 |
| 3 | ET | 强化肿瘤 | 应在 T1C 上呈现增强，是检测任务最关键的病灶信号之一。 |
| 4 | RC | 切除腔 | 主要出现在治疗后病例，不能无依据生成大量孤立空腔。 |

### 2.3 小病灶是重点，不是噪声

Task 文档明确强调：

1. 小于 `27 mm^3` 的小病灶具有临床相关性。
2. 分割指标 Dice/NSD 只应用于大于 `275 mm^3` 的病灶。
3. 检测指标 F1/AUC 会关注每一个独立病灶。

因此，G2 不能采用“删除所有小连通域以提高 Dice”的思路。对 MET 来说，小连通域可能是目标病灶，也可能是假阳性。G2 需要把它们分开评估：

1. 真实小病灶召回是否提升。
2. synthetic data 是否导致假阳性小连通域暴涨。
3. 小于 `27 mm^3`、`27-275 mm^3`、大于 `275 mm^3` 三档 lesion 的表现是否不同。

### 2.4 数据空间混合

当前数据包含 native space、T1C 1 mm^3 共配准空间和 SRI24 空间。不同病例 shape、spacing、affine 差异大。

G2 的所有脚本必须遵守：

1. 不写死 `240x240x155`。
2. 不写死 spacing 为 `1,1,1`。
3. 不提前强行 resample 原始数据。
4. 转 nnU-Net 时优先让 nnU-Net planner/preprocessor 处理 spacing 差异。
5. synthetic case 必须继承或明确记录 source case 的 affine/header。

## 3. 往年方案对今年 G2 的真实启发

### 3.1 2023/2024 GliGAN 的核心不是 GAN，而是局部插入式数据增强

往年 GliGAN 的流程可以抽象为：

```text
真实病例四模态 MRI
  -> 真实或生成的 lesion label patch
  -> 在脑内寻找可插入位置
  -> 更新 segmentation
  -> 对 ROI 四模态进行局部生成/修复
  -> 保存为 synthetic case
```

这对今年 G2 的启发是：

1. synthetic data 应优先保留真实脑背景，减少解剖结构崩坏。
2. G2 应控制 lesion mask、插入位置、标签组合和体积分布。
3. G1 diffusion 应负责 ROI 内四模态病灶影像生成。
4. 保存时应继承 source case 的 affine/header。
5. 每个 synthetic run 都必须有 manifest/log 记录；per-case metadata 可以作为增强项，但第一阶段不强制。

### 3.2 Diffusion 替代 GAN 的合理位置

今年推荐把 diffusion 放在以下位置：

```text
旧 GliGAN:
  label generator + GAN modality generator

今年建议:
  G2 lesion/ROI 控制 + G1 diffusion ROI generator
```

也就是说，G2 不应把 diffusion 当成“万能整脑生成器”，而应先当成“更强的局部病灶生成/修复器”。

### 3.3 3D WDM 的启发

往年 3D WDM/inpainting 的关键思想是：

1. 已知区域保持真实。
2. 只在 mask ROI 内加噪和生成。
3. 采样后再把 ROI 融合回真实图像。
4. 检查 ROI 边界强度连续性。

这非常适合今年 MET 的第一阶段。因为脑转移瘤通常局部小、多发、位置分散，完全生成整脑的收益不一定比局部生成更高，但风险明显更大。

### 3.4 2025 on-the-fly 增强的启发

2025 第一名成人胶质瘤方案强调 on-the-fly 数据增强，不预先保存海量 synthetic NIfTI。这对今年有价值，但不适合作为第一步。

今年建议：

1. 第一阶段：离线 synthetic dataset，方便检查、复现和给 S1/S2 训练。
2. 第二阶段：如果离线合成质量稳定，再考虑把插入逻辑迁移到 nnU-Net dataloader。
3. 第三阶段：结合小病灶、RC 稀有标签、多发病灶做定向 on-the-fly 增强。

## 4. 推荐的生成路线优先级

### 4.1 优先级 1：局部 lesion-conditioned inpainting / insertion

这是第一阶段推荐路线。

流程：

1. 从真实训练病例中选择 source case。
2. 基于真实 lesion 分布或规则生成一个 candidate lesion mask。
3. 在脑内、非原病灶区域、不会越界的位置确定 ROI。
4. 把 lesion mask 插入 segmentation。
5. 把 ROI 和条件信息交给 G1 diffusion。
6. G1 输出 ROI 内四模态 synthetic patch。
7. G2 将 synthetic patch 融合回 source case。
8. G2 保存四模态和更新后的 `seg.nii.gz`。
9. G2 执行 QC。

优点：

1. 保留真实解剖背景。
2. 更容易控制 label。
3. 更容易继承 affine/header。
4. 更容易定位质量问题。
5. 更贴近往年 GliGAN 的成功经验。

风险：

1. ROI 边界可能出现强度断层。
2. G1 如果按 2D 切片生成，z 轴连续性可能差。
3. 插入位置若不合理，会生成医学上不可信的病灶。

### 4.2 优先级 2：真实 label 条件下的局部重生成

这条路线适合模型 smoke test 和质量基准。

流程：

1. 选取真实训练病例。
2. 使用真实 lesion mask 作为条件。
3. 在真实 lesion ROI 内让 G1 diffusion 重生成病灶影像。
4. label 保持真实不变。
5. 对比原图和重生成图像。

用途：

1. 测试 G1 是否能在真实 label 条件下生成合理四模态表现。
2. 测试 ROI 融合、边界连续性和 header 保存。
3. 建立 G2 QC 的 accepted/rejected 样例库。

限制：

1. 它不增加新的 label 分布。
2. 它更像验证生成器是否可靠，不是最终数据增强主力。

### 4.3 优先级 3：2.5D / slice-stitching diffusion

如果 G1 只能做 2D 或 2.5D，则 G2 必须增加 3D 连续性约束。

G2 需要检查：

1. 同一 lesion 在相邻切片上的面积是否平滑变化。
2. lesion 是否在 z 轴突然断裂。
3. ET/SNFH/NETC 是否出现切片级闪烁。
4. 多模态 ROI 是否在同一空间位置一致。
5. 3D 连通域数量是否异常增加。

如果 3D 连续性不过关，2D 生成图像即使单张好看，也不能进入训练。

### 4.4 优先级 4：full-case generation

第一阶段不推荐。

原因：

1. 需要生成全脑解剖结构，难度远高于局部 ROI。
2. 容易破坏真实 MRI 空间、spacing、affine。
3. 标签和图像一致性很难保证。
4. 生成速度和存储成本高。
5. 对下游分割提升未必优于局部插入。

只有在局部方案跑通且 G1 有成熟 3D 全病例生成能力后，才考虑小规模试验。

## 5. 当前阶段的总执行路线

G2 当前应按三条线并行推进。

### 5.1 线一：真实数据工程线

目标：在 G1 模型完成前，把真实数据基础设施全部打好。

按顺序执行：

1. 确认训练集、验证集、corrected labels 的本地路径。
2. 扫描真实训练集，生成 `real_train_manifest_raw.csv`。
3. 扫描 validation，生成 `real_validation_manifest.csv`。
4. 应用 corrected labels，生成 `corrected_label_overlay.csv`。
5. 生成最终 `real_train_manifest.csv`。
6. 标记非法 label 病例。
7. 统计 shape、spacing、dtype、affine hash。
8. 统计 label 值域、类别体积、lesion 数量、lesion 体积分布。
9. 输出 `real_data_qc_summary.md`。
10. 转换 real-only nnU-Net dataset。
11. 生成固定真实验证 fold。
12. 跑 `nnUNetv2_plan_and_preprocess --verify_dataset_integrity`。

### 5.2 线二：G1-G2 契约线

目标：G1 模型还没训练好时，就把后续对接格式定死。

按顺序执行：

1. 写 `G1_G2_diffusion_output_contract.md`。
2. 明确 G1 生成模式：local-insertion、real-label-regeneration、slice-stitching 或 full-case。
3. 明确输入：real manifest、source case、ROI mask、seed、checkpoint。
4. 明确输出：GliGAN-compatible raw 四模态 `.nii.gz`、`seg.nii.gz`、`synthetic_generation_manifest.csv`、`generation_log.jsonl`。
5. 明确模态顺序：`t1n/t1c/t2w/t2f`。
6. 明确 label 值域：`{0,1,2,3,4}`。
7. 明确失败日志：`generation_log.jsonl`。
8. 明确可复现性：同一 seed 和 checkpoint 应得到同一输出。
9. 明确中断续跑规则：已完成病例不重复生成。
10. 明确 smoke test 标准：至少 10 个 synthetic case 能通过 G2 QC。

### 5.3 线三：质量验证线

目标：即使没有 G1 模型，也先把 synthetic data 进入训练前后的评价框架设计清楚。

按顺序执行：

1. 建立 QC 指标清单。
2. 建立 accepted/rejected 的判定规则。
3. 建立 lesion volume 分档：`<27 mm^3`、`27-275 mm^3`、`>275 mm^3`。
4. 建立 label 组合规则：ET-only、ET+SNFH、ET+NETC、ET+SNFH+NETC、RC 等。
5. 建立 teacher model QC 的接口占位。
6. 建立 real-only vs real+synth 的消融模板。
7. 规定第一轮合成比例：`0.25x`、`0.5x`，暂不直接 `1x` 以上。
8. 规定验证集只含真实病例。
9. 规定报告必须同时写成功样本和失败样本。
10. 规定 synthetic data 不提升指标时的回溯方式。

## 6. G1-G2 数据契约

详细契约见：

```text
work_space/G2/docs/G1_G2_diffusion_output_contract.md
```

这里记录调整后的关键原则，避免和独立契约文件冲突。

### 6.1 为什么不直接采用理想契约

`data_space/task1_2026/datasets/数据契约.md` 中的理想契约要求 G1 直接输出 `SYN-MET-000001` 形式的完整病例，并且每例都有 `metadata.json`。这个方向适合作为最终归档规范，但不适合作为第一阶段生成模型对接规范。

原因是当前生成式模型的数据输入输出实际沿用 2023 GliGAN：

1. 用 CSV 记录病例路径、肿瘤中心、bbox 和尺寸。
2. 以 `96x96x96` 局部 ROI 为生成单位。
3. 输入是 noisy MRI crop 与 region-based label crop。
4. 输出是插入到真实病例背景中的四模态 synthetic case。
5. raw synthetic case 命名通常带 source case、`real_label` 或 `fake_label` 信息。
6. 原始 GliGAN 代码没有 per-case `metadata.json`，追踪信息需要由 G2 用 manifest 和 log 承接。

### 6.2 G2 当前采用的三层契约

1. 理想资产层：最终希望所有 synthetic data 都可追踪、可审计、可复现，对应 `SYN-MET-*`、manifest、QC、nnU-Net 导出记录。
2. GliGAN 兼容 raw 层：G1 第一阶段按 2023 GliGAN 风格输出 raw synthetic cases，使用 CSV、`96x96x96` ROI、`real/fake_label` 目录命名。
3. 训练导出层：G2 通过 QC 后把 accepted cases 转成 nnU-Net v2 的 `imagesTr/labelsTr`，并统一模态顺序。

### 6.3 第一阶段 G1 最低交付

G1 第一次 smoke test 不需要直接满足完整理想契约，但至少要交付：

| 项目 | 必需性 | 说明 |
|---|---|---|
| 10-20 个 raw synthetic cases | 必需 | 每例四模态 + `seg`，可由 `nibabel` 读取。 |
| 推理命令或脚本 | 必需 | 能基于 G2 提供的 GliGAN-compatible CSV 批量生成。 |
| checkpoint 信息 | 必需 | 记录 diffusion checkpoint、label generator checkpoint、版本号。 |
| `generation_log.jsonl` | 必需 | 成功、失败、跳过都要记录。 |
| `synthetic_generation_manifest.csv` | 建议 G1 生成，G2 可补建 | 第一版不强制每例 `metadata.json`，但不能没有 manifest/log。 |
| label channel 说明 | 必需 | 说明使用 4 通道 TC/WT/ET/RC，还是临时 3 通道无 RC。 |
| 生成模式说明 | 必需 | `single_modal_gligan` 或 `multi_modal_joint`。 |
| intensity 说明 | 必需 | 是否按 GliGAN crop 内归一化到 `[-1,1]`，是否使用 extended Gaussian noise。 |

### 6.4 模态映射

G1 内部可以继续使用 GliGAN key：

```text
scan_t1ce -> 2026 t1c
scan_t1   -> 2026 t1n
scan_t2   -> 2026 t2w
scan_flair -> 2026 t2f
label     -> 2026 seg
```

G2 最终导出给 S1/S2 时必须使用 nnU-Net 通道顺序：

```text
0000 = t1n
0001 = t1c
0002 = t2w
0003 = t2f
```

### 6.5 raw synthetic 命名

第一阶段 raw output 允许使用 GliGAN 兼容命名：

```text
<source_case_id>_<real|fake>_label_<label_index>
```

示例：

```text
BraTS-MET-01094-003_fake_label_0
BraTS-MET-01184-002_real_label_0
```

G2 在下游转换时再分配唯一 synthetic ID，例如 `SYN-MET-000001`。这样既兼容旧代码，也避免 synthetic case 在训练集中和真实病例 ID 混淆。

## 8. 真实数据 manifest 方案

### 8.1 为什么先做 manifest

manifest 是 G2 后续所有工作的根。没有 manifest，数据转换、QC、合成来源追踪和消融分析都会变成手工记忆。

manifest 能回答：

1. 这个病例来自训练集还是验证集？
2. 它有哪些模态？
3. label 是原始 label 还是 corrected label？
4. shape/spacing/affine 是否一致？
5. 是否有非法 label？
6. 是否可以进入第一轮训练？
7. synthetic case 是从哪个真实病例派生的？

### 8.2 real_train_manifest 字段

```text
case_id
split_source
case_dir
t1n_path
t1c_path
t2w_path
t2f_path
raw_seg_path
effective_seg_path
label_source
shape
spacing
affine_hash
image_dtypes
label_dtype
labels_present
has_illegal_label
illegal_label_values
has_corrected_label
qc_pass
qc_reason
```

字段解释：

1. `case_id`：如 `BraTS-MET-00001-000`。
2. `split_source`：如 `batch1_top_level` 或 `ucsd_training`。
3. `raw_seg_path`：原始 label 路径。
4. `effective_seg_path`：训练实际使用的 label 路径；如果有 corrected label，则指向 corrected label。
5. `label_source`：`raw`、`corrected`、`excluded`。
6. `affine_hash`：用于快速发现 affine 不一致。
7. `qc_pass`：是否进入第一轮训练。
8. `qc_reason`：如果不通过，写明原因。

### 8.3 corrected labels 处理

当前 corrected labels：

```text
BraTS-MET-01094-003-seg.nii.gz
BraTS-MET-01184-002-seg.nii.gz
```

处理规则：

1. 不建议直接覆盖原始训练集。
2. 在 manifest 中建立逻辑覆盖层。
3. 对每个 corrected label 校验 shape、spacing、affine 是否与 source case 一致。
4. 输出 `corrected_label_overlay.csv`。
5. 报告中必须写出 corrected label 应用清单。

### 8.4 非法 label 处理

当前已知风险：

1. `BraTS-MET-01184-002` 含 label value `8`，有 corrected label 可修正。
2. `BraTS-MET-01094-002` 含 label value `6`，当前没有对应 corrected label。

推荐第一轮处理：

1. `BraTS-MET-01184-002` 使用 corrected label。
2. `BraTS-MET-01094-002` 标记为 `qc_pass=false`。
3. 不静默执行 `6 -> 某标签` 的映射。
4. 若后续确认映射规则，写入 `label_fix_rules.yaml`，并在报告中说明。

## 9. lesion 分布统计与 synthetic 目标分布

### 9.1 必须统计的真实数据分布

G2 在生成 synthetic data 前，必须先统计真实训练集：

1. 每例 lesion 数量。
2. 每个 connected component 的体积。
3. 小于 `27 mm^3` 的 lesion 数量。
4. `27-275 mm^3` 的 lesion 数量。
5. 大于 `275 mm^3` 的 lesion 数量。
6. 每类标签体积：NETC、SNFH、ET、RC。
7. 每类标签出现病例数。
8. label 组合：ET-only、ET+SNFH、ET+NETC、ET+SNFH+NETC、RC 等。
9. 多发病灶病例比例。
10. 空 mask 或极小 mask 病例比例。

### 9.2 synthetic target distribution

第一轮 synthetic data 不应追求完全复刻真实分布，而应有明确目的：

1. 补充小病灶。
2. 补充多发病灶。
3. 补充稀有 RC。
4. 补充 ET/SNFH/NETC 合理组合。
5. 控制假阳性风险。

建议第一轮目标：

1. 生成 100-300 个候选 synthetic cases。
2. accepted synthetic 不超过真实训练样本的 25%。
3. 小病灶比例高于真实训练集，但不要全部是小病灶。
4. 每个 source case 第一轮最多生成 1 个 synthetic case。
5. 不使用 validation source case。
6. RC 类 synthetic 先保守生成，优先使用真实 RC source case 或真实 RC label 变体。

### 9.3 lesion mask 生成策略

第一轮 mask 来源按优先级：

1. 真实训练集 lesion mask 裁剪后平移/缩放。
2. 真实小病灶 mask 轻微形变。
3. 多 lesion 组合插入。
4. 规则生成的椭球/不规则 mask。
5. 完全由模型生成的 label。

推荐第一轮不要依赖纯模型生成 label。label 质量一旦失控，后续 diffusion 图像再好也没有训练价值。

## 10. synthetic QC 方案

### 10.1 强制拒绝条件

满足任一条件即拒绝：

1. 缺少任一模态。
2. 缺少 `seg.nii.gz`。
3. 四模态与 label shape 不一致。
4. 四模态与 label affine/spacing 明显不一致。
5. 图像含 NaN/Inf。
6. label 含 `{0,1,2,3,4}` 之外的值。
7. label 不是整数或无法安全转换为整数。
8. label 全空，且 manifest 未显式允许空 mask。
9. lesion 全部落在脑外或零背景区。
10. ROI 边界出现明显强度断层。
11. 2D/slice-stitching 结果在 z 轴严重断裂。
12. 缺少 `synthetic_generation_manifest.csv` 或 `generation_log.jsonl`，且无法从目录结构补建。

### 10.2 记录但不立即拒绝的指标

1. lesion 数量。
2. 每类 label 体积。
3. 每个 connected component 体积。
4. 小病灶分档数量。
5. ET/SNFH/NETC/RC 组合。
6. 各模态脑区强度分位数。
7. ROI 内外强度差。
8. source 与 synthetic 的相似度。
9. teacher prediction 与 synthetic label 的 Dice。
10. teacher lesion count 与 label lesion count 差异。

### 10.3 图像 QC

每个 synthetic case 至少检查：

1. t1n/t1c/t2w/t2f 是否均可读取。
2. 非零脑区比例是否合理。
3. ROI 边界内外强度是否平滑。
4. T1C 中 ET 区域是否有增强趋势。
5. T2F/FLAIR 中 SNFH 区域是否有高信号趋势。
6. 四模态 lesion 位置是否一致。
7. 是否出现棋盘格、条纹、切片闪烁、边缘块状伪影。

### 10.4 3D 连续性 QC

如果 G1 使用 2D 或 2.5D diffusion，G2 必须额外检查：

1. 每个 lesion 的 z 轴连续切片数。
2. 相邻切片 lesion 面积变化率。
3. lesion 是否一层有、一层无地闪烁。
4. 每类 label 的 connected component 数量是否异常。
5. 小病灶是否变成大量孤立单 voxel。

### 10.5 人工抽查

第一轮每个 generation run 必须人工抽查：

1. accepted 样本至少 20 例。
2. rejected 样本至少 20 例。
3. 每类 reject reason 至少看 3 例。
4. 小病灶样本至少看 10 例。
5. RC 样本若生成，全部抽查。

人工抽查应输出 montage，不只看单张切片。

## 11. nnU-Net 转换方案

### 11.1 命名规则

使用 nnU-Net v2 多模态标准：

```text
Dataset260_BraTS2026_MET_RealOnly/
  imagesTr/
    BraTSMET_000001_0000.nii.gz  # t1n
    BraTSMET_000001_0001.nii.gz  # t1c
    BraTSMET_000001_0002.nii.gz  # t2w
    BraTSMET_000001_0003.nii.gz  # t2f
  labelsTr/
    BraTSMET_000001.nii.gz
  dataset.json
```

通道顺序固定：

1. `0000`: t1n
2. `0001`: t1c
3. `0002`: t2w
4. `0003`: t2f

### 11.2 dataset.json

标准 label 版本：

```json
{
  "channel_names": {
    "0": "t1n",
    "1": "t1c",
    "2": "t2w",
    "3": "t2f"
  },
  "labels": {
    "background": 0,
    "NETC": 1,
    "SNFH": 2,
    "ET": 3,
    "RC": 4
  },
  "file_ending": ".nii.gz"
}
```

是否使用 region-based training 由 S1/S2 决定。G2 第一版先交付标准 label 版本，避免过早引入额外变量。

### 11.3 数据集版本

建议至少准备三组：

1. `Dataset260_BraTS2026_MET_RealOnly`：只含真实训练数据。
2. `Dataset261_BraTS2026_MET_RealSynth025`：真实数据 + 0.25x accepted synthetic。
3. `Dataset262_BraTS2026_MET_RealSynth050`：真实数据 + 0.5x accepted synthetic。

不要第一轮直接做 `1x` 或 `>1x` synthetic。

## 12. 下游消融验证

### 12.1 最小验证闭环

第一轮只需要跑通最小闭环：

1. 固定一个真实验证 fold。
2. 训练 real-only nnU-Net baseline。
3. 训练 real + 0.25x synthetic。
4. 训练 real + 0.5x synthetic。
5. 保持训练配置一致。
6. 对比真实验证集指标。

### 12.2 必须比较的指标

1. Dice。
2. NSD。
3. lesion-wise F1。
4. lesion-wise AUC。
5. 小于 `27 mm^3` lesion recall。
6. `27-275 mm^3` lesion recall。
7. 大于 `275 mm^3` lesion segmentation quality。
8. 假阳性 connected components 数。
9. ET/NETC/SNFH/RC 分类别表现。

### 12.3 结果判定

synthetic data 被认为有效，需要至少满足：

1. 总体 Dice/NSD 不显著下降。
2. lesion-wise F1 或 AUC 有提升。
3. 小病灶 recall 有提升或不下降。
4. 假阳性数量可控。
5. 对至少一个低频或困难类别有明确收益。

如果只提升 Dice，但检测 F1/AUC 下降，则不能认为成功。

### 12.4 失败回溯

如果 synthetic 版本性能下降，按以下顺序排查：

1. synthetic 比例是否过高。
2. 是否有某个 generator checkpoint 通过率低。
3. 是否小病灶假阳性过多。
4. 是否 label 与图像不一致。
5. 是否某类 label 组合不合理。
6. 是否 source case 与 validation 存在同源泄漏。
7. 是否 nnU-Net 转换通道顺序错误。
8. 是否 preprocessing 与 baseline 不一致。

1. 生成 100-300 个候选 synthetic cases。
2. 自动 QC。
3. 人工抽查。
4. 输出 accepted/rejected manifest。
5. 准备 0.25x 和 0.5x synthetic 数据集。

### M3：一折消融，3-7 天

1. S1/S2 训练 real-only baseline。
2. S1/S2 训练 real + 0.25x synthetic。
3. S1/S2 训练 real + 0.5x synthetic。
4. G2 汇总指标。
5. G2 写第一轮 synthetic quality report。

1. 根据消融结果调整 lesion 目标分布。
2. 调整 QC 阈值。
3. 引入 teacher model QC。
4. 扩大合成规模。
5. 评估是否进入 on-the-fly 阶段。

## 16. 关键判断

1. G2 方案不需要大改方向，需要调整优先级。
2. diffusion 替代 GAN 的第一落点应是局部 ROI 生成，不是整脑生成。
3. G2 的强项应该是数据系统、质量控制和下游验证，而不是盲目生成更多数据。
4. 当前阶段 G2 完全可以独立推进大量工作，不必等待 G1 模型训练完成。
5. 第一轮成功标准是跑通小规模闭环，而不是生成规模。
6. 真正决定 synthetic data 价值的是真实验证集消融，不是生成样例图。
