## 1. 结论先行

G2 的实现难度为偏高，建议按 7.5/10 估计。如果 G1 已经给出稳定的病例级生成器，并且输出天然包含四模态 MRI 与 0-4 整数标签，G2 的主要工作是工程化、格式转换、质量控制和下游验证，难度约 6.5-7/10。如果 G1 只给出切片式条件潜在扩散模型，G2 还需要负责切片到 3D 病例的拼接一致性、标签几何一致性、跨模态强度一致性与批量失败恢复，难度会上升到 8-8.5/10。

G2 最关键的判断不是“生成多少数据”，而是“生成的数据是否能提高真实验证集上的分割和检测表现”。BraTS 2026 Task 1 加入检测排行榜，并且强调小病灶，因此只用 FID、MS-SSIM 这类图像相似指标是不够的。G2 的最终质量标准应以 S 组固定验证划分上的消融结果为准：真实数据训练 vs 真实数据加合成数据训练，比较 Dice、NSD、病灶检测 F1/AUC，尤其要单独看小病灶和多发病灶。

建议 G2 采用“双轨制”：

1. 离线合成数据集：先交付可追踪、可复现、可给 nnU-Net 直接训练的 `imagesTr + labelsTr` 数据。
2. 可选的 on-the-fly 生成接口：借鉴 2025 第一名成人胶质瘤方案，把合成肿瘤动态插入训练 patch，减少 3D 数据落盘压力。

## 3. BraTS 2026 Task 1 对 G2 的约束

Task 1 是脑转移瘤检测与分割，目标算法要适用于治疗前和治疗后病例。

四标签体系：

| 标签 | 缩写 | 含义 |
|---:|---|---|
| 1 | NETC | 非强化肿瘤核心 |
| 2 | SNFH | 周围非强化 FLAIR 高信号，包括水肿和浸润组织 |
| 3 | ET | 强化肿瘤 |
| 4 | RC | 切除腔 |

G2 需要特别注意三点：

1. 2026 年新增检测排行榜。病灶检测按 lesion 评估，指标包括 F1 和多个阈值下 F1 的 AUC。
2. 小病灶很重要。Task 文档特别提到小于 27 mm^3 的病灶具有临床相关性；分割指标只应用于大于 275 mm^3 的病灶，但检测应覆盖独立病灶。
3. 数据空间混合。数据包含 native space、T1C 1 mm^3 共配准空间和 SRI24 空间。G2 的合成数据不能假设所有病例都是同一 shape、同一 affine、同一 spacing。

因此，G2 的合成数据不能只追求“看起来像 MRI”。必须同时满足：

- 标签结构符合 0-4 标签系统。
- lesion 数量、体积、空间分布接近真实训练集。
- 多模态之间有一致的病灶表现。
- affine/header/spacing 不破坏 nnU-Net 和后续评估。
- 合成样本不会泄漏验证集或测试集信息。

## 4. 冠军方案对 G2 的启发

### 4.1 2023 第一名成人胶质瘤：GliGAN + 注册增强 + 多模型集成

本地论文：`往年文章和code/2023/2023年第一名.pdf`
本地代码：`往年文章和code/2023/Segmentation_Tasks/GliGAN/`
论文链接：https://arxiv.org/abs/2402.17317

核心做法：

- 使用 GAN 和配准显著扩大训练数据。
- 训练三个分割体系：标准 nnU-Net、Swin UNETR、BraTS 2021 winner 变体。
- 使用 5 折交叉验证，总计 45 个 checkpoint 做集成。
- 生成侧包含两个部分：
  - label generator：生成或采样合成标签。
  - modality generator：针对 T1、T1Gd/T1C、T2、FLAIR 分别生成对应影像。
- 代码里有大规模合成入口：
  - `main_random_label_random_dataset_generator_multiprocess.py`
  - 支持 `start_case/end_case` 切分任务。
  - 支持 `new_n` 控制每个真实病例生成多少合成病例。
  - 保存 NIfTI 时会继承参考图像的 affine/header。

对 G2 的直接启发：

1. 数据生成要有 manifest，而不是只落一堆 NIfTI。每个合成病例需要记录来源病例、随机种子、使用真实标签还是合成标签、生成器 checkpoint、失败原因和 QC 结论。
2. 合成数据不能进入验证 fold。2023/2024 方案明确保持验证集只有真实病例，并避免由验证病例派生出的合成样本进入训练。
3. 病灶级后处理很重要。冠军方案为了 lesion-wise 评估做了小连通域阈值处理。对 2026 MET，阈值不能粗暴照搬，因为小病灶检测本身是目标，应把“检测敏感”和“假阳性控制”分开评估。

### 4.2 2025 第一名成人胶质瘤：on-the-fly GliGAN 增强

本地论文：`往年文章和code/2025/2025年第一名.pdf`
论文链接：https://arxiv.org/abs/2509.24973

核心做法：

- 不预先存储巨量 3D 合成病例，而是在 nnU-Net 训练过程中动态插入合成肿瘤。
- 使用已有 GliGAN 权重和肿瘤插入策略。
- 对小病灶和类别不平衡做定向增强，例如缩放真实 label、调整类别组合、增加 ET/NETC 的出现频率。
- 训练 baseline、regular on-the-fly、custom on-the-fly 三个模型，再做集成。

对 G2 的直接启发：

1. 如果离线生成数据太慢或占用空间太大，G2 应给 S1/S2 预留 on-the-fly 接口。
2. 质量控制不应只在生成后做，也可以在训练采样前做。比如只允许通过 QC 的 label mask 参与动态插入。
3. 对 2026 MET，custom augmentation 的重点应改为：
   - 增加小于 275 mm^3 和小于 27 mm^3 附近的病灶样本。
   - 增加多发病灶病例。
   - 控制 RC、NETC、ET 的结构组合，避免生成医学上不合理的空洞或孤立标签。

### 4.3 2024 Synthesis Task 7/8：3D WDM 扩散模型

本地说明：

- `往年文章和code/2023/BraTS2024_Task7.md`
- `往年文章和code/2023/BraTS2024_Task8.md`

本地代码：

- `往年文章和code/2023/Synthesis/Task8/wdm-3d/`
- `往年文章和code/2023/Synthesis/Task8/testing/inference.py`

挑战论文链接：https://arxiv.org/abs/2305.08992

核心做法：

- 使用 3D Wavelet Diffusion Model。
- inpainting 任务中，已知区域在每一步采样中保持为原图，只生成 ROI。
- missing MRI 任务中，已知 3 个模态保持不加噪，只生成缺失模态。
- 采样步数很大，示例中常用 3000-5000 steps，推理很慢。

对 G2 的直接启发：

1. 如果 G1 用扩散模型做局部肿瘤生成，最好保持真实背景不动，只在 ROI 内替换，这比完全生成整脑病例更稳。
2. G2 的 QC 要检查 ROI 边界处的强度连续性，避免肿瘤插入边界出现明显断层。
3. 如果 G1 是切片式 latent diffusion，G2 要额外检查跨切片连贯性。单张 2D 切片好看，不代表 3D lesion 合理。

## 5. G2 的推荐技术路线

### 阶段 0：和 G1 定义生成输出契约

G1 必须给 G2 一个稳定输出规范。推荐每个合成病例目录如下：

```text
synthetic_raw/
  SYN-MET-000001/
    SYN-MET-000001-t1n.nii.gz
    SYN-MET-000001-t1c.nii.gz
    SYN-MET-000001-t2w.nii.gz
    SYN-MET-000001-t2f.nii.gz
    SYN-MET-000001-seg.nii.gz
    metadata.json
```

`metadata.json` 至少包含：

- `case_id`
- `source_case_id`
- `generator_checkpoint`
- `label_source`: real_label, synthetic_label, hybrid_label
- `seed`
- `spacing`
- `shape`
- `affine_source`
- `generation_mode`: full_case, local_insertion, slice_stitching
- `lesion_count`
- `label_volumes_mm3`
- `qc_status`
- `qc_reject_reason`

G2 应明确告诉 G1：没有 `seg.nii.gz` 的合成图像对分割训练价值有限，除非 S 组另有半监督方案。G2 的主交付应以“图像 + 标签”成对数据为准。

### 阶段 1：建立数据 manifest

建议先写一个 manifest，而不是直接转换：

```text
manifests/
  real_train_manifest.csv
  synthetic_manifest.csv
  accepted_synthetic_manifest.csv
  rejected_synthetic_manifest.csv
```

字段建议：

- `case_id`
- `is_synthetic`
- `source_case_id`
- `t1n_path`
- `t1c_path`
- `t2w_path`
- `t2f_path`
- `seg_path`
- `shape`
- `spacing`
- `affine_hash`
- `labels_present`
- `n_lesions`
- `min_lesion_volume_mm3`
- `max_lesion_volume_mm3`
- `qc_pass`
- `qc_reason`

manifest 的价值是后面可以追溯每一个进入训练的样本，不然一旦 synthetic data 造成 S 组性能下降，很难定位是某个生成批次、某个 checkpoint，还是某类标签出了问题。

### 阶段 2：转换为 nnU-Net v2 格式

推荐使用 nnU-Net v2 标准多模态命名：

```text
Dataset26X_BraTS2026_MET_RealSynth/
  imagesTr/
    BraTSMET_000001_0000.nii.gz  # t1n
    BraTSMET_000001_0001.nii.gz  # t1c
    BraTSMET_000001_0002.nii.gz  # t2w
    BraTSMET_000001_0003.nii.gz  # t2f
  labelsTr/
    BraTSMET_000001.nii.gz
  dataset.json
```

建议通道顺序：

```json
{
  "channel_names": {
    "0": "t1n",
    "1": "t1c",
    "2": "t2w",
    "3": "t2f"
  },
  "labels": {
    "background": 0,
    "NETC": 1,
    "SNFH": 2,
    "ET": 3,
    "RC": 4
  },
  "file_ending": ".nii.gz"
}
```

如果 S1/S2 决定使用 region-based training，可另建一个 region 版本：

```json
{
  "labels": {
    "background": 0,
    "whole tumor": [1, 2, 3],
    "tumor core": [1, 3],
    "enhancing tumor": 3,
    "resection cavity": 4
  },
  "regions_class_order": [1, 2, 3, 4]
}
```

注意：不要机械照抄 `往年文章和code/2023/Segmentation_Tasks/example/dataset_2024_glioma.json`。2026 MET 的官方标签顺序应以当前 Task1 文档为准，尤其是 NETC 和 SNFH 的标签编号。

转换脚本应自动检查：

- 四个模态和 label 是否都存在。
- 四个模态 shape、affine、spacing 是否一致。
- label 是否为整数，取值是否属于 `{0,1,2,3,4}`。
- 是否有 NaN/Inf。
- label 是否和脑区大致重叠。
- NIfTI header 是否可被 nibabel 正常读取。
- 输出文件名是否符合 nnU-Net v2 的 `_0000/_0001/_0002/_0003` 规则。

### 阶段 3：基础 QC

每个合成病例必须先过基础 QC，再进入训练集。

强制拒绝条件：

- 任一模态缺失。
- 任一模态与 label shape 不一致。
- label 含非法值。
- 图像含 NaN/Inf。
- label 全空，但 metadata 未标注为空掩膜病例。
- affine 明显错误，例如全零、奇异矩阵、spacing 为 0。
- 肿瘤完全落在脑外或图像零背景区。
- 合成 ROI 边界出现明显强度断层。

建议统计但不立即拒绝：

- lesion 数量。
- 每类标签体积。
- 每个连通域体积。
- 小于 27 mm^3、27-275 mm^3、大于 275 mm^3 的 lesion 分布。
- ET/NETC/RC/SNFH 的组合情况。
- 图像非零区域强度分位数，例如 p1/p50/p99。
- synthetic 和 source case 的相似度，避免重复度太高。

### 阶段 4：分布 QC

G2 应先用真实训练集建立分布基线，再筛合成数据。没有训练集时，只能做格式和图像层 QC，不能完整判断质量。

建议分布对齐项：

1. 病灶数量分布：单发、多发、极多发。
2. 病灶体积分布：重点覆盖小病灶，不能只生成大肿瘤。
3. 类别体积分布：ET、NETC、SNFH、RC 的出现频率。
4. 空间位置分布：脑实质内位置、皮层附近、小脑/脑干等区域。
5. 强度分布：每个模态在脑区、病灶区、病灶周边的强度分位数。
6. 跨模态一致性：同一 lesion 在 T1C、FLAIR、T2W、T1N 的表现是否合理。

建议输出图：

- lesion volume histogram。
- label class frequency bar plot。
- per-modality intensity percentile boxplot。
- real vs synthetic t-SNE/UMAP feature plot。
- synthetic accepted/rejected case montage。
- 每个 generator checkpoint 的通过率和拒绝原因统计。

### 阶段 5：图像相似度与多样性指标

可以做 FID 和 MS-SSIM，但要谨慎解释。

FID 建议：

- 不建议直接用 ImageNet Inception FID 评价 3D MRI。
- 更合理的是使用医学影像特征：
  - 预训练 3D ResNet/MedicalNet。
  - nnU-Net encoder feature。
  - Swin UNETR encoder feature。
  - BraTS MET 预训练分割模型的 bottleneck feature。
- 可按 2D 切片计算作为快速版本，但报告里必须说明它不能完整代表 3D 结构。

MS-SSIM 建议：

- 用于检测多样性不足和近重复。
- synthetic-synthetic MS-SSIM 太高，说明模式坍塌或复制。
- real-synthetic MS-SSIM 太高，说明可能过度贴近 source case。
- MS-SSIM 太低也不一定好，可能表示结构不真实。

G2 报告里应把这些指标列为辅助指标，而不是最终判断。

### 阶段 6：教师模型一致性 QC

如果可用，推荐用 2025/2023 MET 顶尖分割算法或 S1 baseline 跑合成数据：

1. 输入 synthetic MRI。
2. 得到 teacher prediction。
3. 与 synthetic label 比较：
   - label-wise Dice。
   - region-wise Dice。
   - lesion count difference。
   - false positive / false negative components。
4. 过滤 teacher 完全不认可的合成病例。

注意：teacher 模型不是真值。它只能发现明显异常，不能替代人工抽查和真实验证集消融。

### 阶段 7：下游验证闭环

最终有效性必须通过 S 组模型验证。推荐最小闭环：

1. 固定一个真实训练/验证划分。验证集只包含真实病例。
2. 训练 S1 nnU-Net baseline：真实训练数据。
3. 训练 S1 nnU-Net synthetic version：真实训练数据 + 通过 QC 的合成数据。
4. 保持所有参数一致：
   - fold 相同。
   - patch size 相同。
   - epoch/iteration 相同。
   - preprocessing 相同。
   - postprocessing 相同。
5. 比较：
   - Dice。
   - NSD。
   - lesion-wise F1/AUC。
   - 小病灶召回。
   - ET/NETC/RC/SNFH 单项表现。
6. 如果 synthetic 版本下降，按 manifest 回溯：
   - 合成比例是否过高。
   - 某个 checkpoint 生成质量是否差。
   - 小病灶是否假阳性太多。
   - label 与图像是否不一致。

第一轮合成比例建议保守：

- `real : synthetic = 1 : 0.25`
- `real : synthetic = 1 : 0.5`
- 之后再尝试 `1 : 1`

不建议一开始就用每例 10-20 个合成病例淹没真实数据。2023 方案可以大规模合成，但 2026 MET 的小病灶检测对假阳性更敏感。

## 6. G2 模块拆解与难度

| 模块 | 难度 | 主要风险 | 建议 |
|---|---:|---|---|
| 生成任务调度 | 中 | GPU 推理慢，失败恢复困难 | 统一 manifest、seed、checkpoint、日志 |
| NIfTI 合法性检查 | 中 | affine/spacing/header 错误导致 nnU-Net 静默异常 | 用 nibabel 自动检查，保留 source header |
| nnU-Net v2 转换 | 中 | 通道顺序、命名、label mapping 出错 | 写单元测试和 `nnUNetv2_plan_and_preprocess --verify_dataset_integrity` |
| label 几何 QC | 高 | 生成的 lesion 不合理、小病灶假阳性多 | 连通域、体积分布、脑 mask 内约束 |
| 图像质量 QC | 高 | MRI 看似合理但多模态不一致 | 强度分布、ROI 边界、跨模态检查 |
| FID/MS-SSIM | 中 | 指标容易被误用 | 仅作为辅助，多用医学特征 |
| 教师模型 QC | 中高 | Docker/GPU 依赖，teacher 偏差 | 先用 S1 baseline，后接 BrainLesion 容器 |
| 下游消融 | 高 | 训练耗时，变量难控 | 先一折小实验，严格固定划分 |
| 报告撰写 | 中 | 只报好看的图，不报失败样本 | 报通过率、拒绝原因、消融结果 |

## 8. 里程碑建议

### M0：准备阶段，1-2 天

- 获取官方训练集。
- 固定真实验证划分。
- 和 G1 确认输出契约。
- 生成 5-10 个 smoke synthetic cases。

### M1：转换与基础 QC，2-3 天

- 完成 manifest。
- 完成 nnU-Net v2 转换脚本。
- 完成 NIfTI/header/label 检查。
- 跑通 `nnUNetv2_plan_and_preprocess --verify_dataset_integrity`。

### M2：第一版批量生成，3-5 天

- 生成 100-300 个候选合成病例。
- 自动 QC 筛选。
- 输出第一版 `accepted_synthetic_manifest.csv`。
- 做人工抽查。

### M3：S1 一折消融，3-7 天，取决于 GPU

- 真实数据 baseline。
- 真实 + 0.25x synthetic。
- 真实 + 0.5x synthetic。
- 比较验证指标。

### M4：迭代优化，1-2 周

- 根据 S1 结果调整生成比例、label 分布、小病灶策略。
- 引入 teacher model QC。
- 扩大生成规模。
- 完成 G2 质量报告。

## 9. 对 G2 的关键提醒

1. 先别追求大规模，先保证 20 个合成病例能被 nnU-Net 完整读取、训练、反推。
2. 不要用验证集派生合成训练样本。当前本地只有 validation zip，训练前必须获取官方 training zip。
3. 不要把 FID/MS-SSIM 当最终质量指标。BraTS 2026 的核心是分割与检测排名。
4. 不要照搬成人胶质瘤的阈值后处理。MET 小病灶检测敏感，粗暴删除小连通域可能伤害检测榜。
5. 不要混淆模态命名。当前官方文件名使用 `t1n/t1c/t2w/t2f`，而旧代码常写 `t1/t1ce/t2/flair`。
6. 不要混淆标签编号。以 2026 Task1 文档为准：1 NETC, 2 SNFH, 3 ET, 4 RC。
7. 如果 G1 是切片式模型，必须做 3D 连续性 QC。否则生成切片看起来正常，3D lesion 可能抖动或断裂。
8. 合成数据应有版本号。每次生成都记录 checkpoint、seed、代码 commit 和 QC 规则版本。

## 10. 建议脚本清单

后续实现时，G2 可以按以下脚本组织：

```text
tools/g2_build_manifest.py
tools/g2_generate_batch.py
tools/g2_qc_synthetic.py
tools/g2_convert_to_nnunet.py
tools/g2_make_splits.py
tools/g2_teacher_qc.py
tools/g2_feature_metrics.py
tools/g2_report.py
```

每个脚本建议只做一件事：

- `g2_build_manifest.py`：扫描真实/合成数据，输出 CSV。
- `g2_generate_batch.py`：调用 G1 推理接口，生成 raw synthetic cases。
- `g2_qc_synthetic.py`：基础 QC、label QC、强度 QC。
- `g2_convert_to_nnunet.py`：转换到 nnU-Net v2 raw dataset。
- `g2_make_splits.py`：生成固定 fold，确保 validation 只含真实病例。
- `g2_teacher_qc.py`：调用 S1 或 BrainLesion MET segmenter 做一致性筛查。
- `g2_feature_metrics.py`：计算 FID/MS-SSIM/医学特征距离。
- `g2_report.py`：汇总 markdown 报告和图表。

## 11. 推荐优先级

最高优先级：

1. 拿到训练集。
2. 统一 G1 输出契约。
3. 跑通 10 个合成病例到 nnU-Net v2 的完整链路。
4. 固定真实验证划分，避免数据泄漏。

第二优先级：

1. label 几何 QC。
2. 小病灶分布控制。
3. 下游一折消融。
4. teacher model 一致性筛查。

第三优先级：

1. FID/MS-SSIM。
2. 大规模生成。
3. on-the-fly 接口。
4. 多模型联合消融。
