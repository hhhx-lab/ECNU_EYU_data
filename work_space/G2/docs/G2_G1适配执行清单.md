# G2 适配 G1 执行清单

更新日期：2026-06-13
目的：把 G1 的 GliGAN-compatible diffusion raw output 变成 G2 可审计、可 QC、可训练、可报告的数据资产。

## 0. 执行原则

1. G1 负责生成 raw synthetic case。
2. G2 负责接收、记忆化、标准化、QC、导出和报告。
3. 第一阶段不要求 G1 直接输出 `SYN-MET-*`。
4. 第一阶段不要求 G1 大改模型为整脑生成。
5. 任何不可追溯的 synthetic case 都不能进入训练。

## 1. G1 生成前，G2 必须准备

1. 确认 `real_train_manifest.csv` 已更新。
2. 确认 fixed real validation fold 已锁定。
3. 确认 official validation 不进入 source。
4. 确认 corrected label overlay 已应用。
5. 确认非法 label 病例已排除。
6. 确认 `g1_gligan_source_cases_v1.csv` 只包含 allowed source。
7. 确认 `usable_for_gligan96=True` 的病例才用于第一阶段 ROI 生成。
8. 把 `g1_gligan_source_cases_v1.csv` 发给 G1。
9. 同步 G1 必须记录的 run 信息：checkpoint、seed、label_channels、noise_type、sampling method。
10. 约定 G1 smoke run 数量：10-20 cases。

## 2. G1 输出后，G2 第一步 intake

1. 扫描 run 目录。
2. 记录 `generation_run_id`。
3. 检查是否有 `generation_config.json`。
4. 检查是否有 `generation_log.jsonl`。
5. 检查是否有 `synthetic_generation_manifest.csv`。
6. 若缺 manifest，按 `synthetic_generation_manifest_template_g1.csv` 补建。
7. 从 raw case 目录名解析 `source_case_id`、`label_kind`、`label_index`。
8. 识别 legacy suffix：`scan_t1/scan_t1ce/scan_t2/scan_flair`。
9. 映射为 2026 suffix：`t1n/t1c/t2w/t2f`。
10. 生成 `synthetic_candidate_manifest_{run_id}.csv`。

执行入口：

```bash
python work_space/G2/code/g2_synthetic_raw_intake_qc.py \
  --synthetic-run-root /path/to/G1/run_id \
  --synthetic-run-id run_id
```

## 3. G2 自动 QC

按顺序执行：

1. L0 raw delivery intake。
2. L1 四模态和 `seg` 完整性。
3. L2 NIfTI 可读性、shape、spacing、affine。
4. L3 图像数值和 label 合法性。
5. L4 source 与 validation leakage。
6. L5 G1 ROI 插入一致性。
7. L6 lesion-level 结构。
8. L7 多模态医学一致性。
9. L8 扩散生成痕迹。
10. L9 teacher model 辅助。
11. L10 batch distribution。
12. L11 nnU-Net integrity。
13. L12 real validation ablation。

输出：

```text
work_space/G2/results/qc/qc_metrics_{run_id}.csv
work_space/G2/results/qc/diffusion_quality_metrics_{run_id}.csv
```

## 4. 强制拒绝条件

出现以下任一情况，单例不能进入训练：

1. source 来自 fixed real validation fold。
2. source 来自 official validation。
3. source 缺失且无法恢复。
4. 缺任一模态或缺 `seg`。
5. NIfTI 无法读取。
6. shape/spacing/affine 在同一病例内不一致。
7. 图像含 NaN/Inf。
8. 图像是常数图。
9. label 非整数。
10. label 值域超出 `{0,1,2,3,4}`。
11. lesion 大面积在脑外。
12. 严重 ROI 方块伪影。
13. 严重 z 轴断裂。
14. 缺 checkpoint/seed/source 等关键可复现字段。

## 5. 人工复查触发

以下情况不能自动 accepted：

1. raw case 命名无法解析。
2. ROI 坐标缺失。
3. 非 ROI 区域被明显改动。
4. tiny lesion ratio 过高。
5. lesion 数超过真实分布高分位。
6. RC synthetic 不是来自明确 RC 语境。
7. ET 在 t1c 上没有合理增强。
8. SNFH 在 t2f/t2w 上没有合理高信号语境。
9. teacher model 与 label 严重不一致。
10. synthetic 与某个真实 ROI 过度相似，疑似复制。

## 6. 单例质量分级

| 等级 | 名称 | 处理 |
|---|---|---|
| A | excellent | 进入训练，可作为展示样例 |
| B | accepted | 进入训练 |
| C | ablation_only | 只进受控消融 |
| D | needs_regeneration | 返回 G1 重跑 |
| F | rejected | 不使用 |

## 7. 批次放行标准

1. validation leakage 数为 0。
2. NIfTI 可读率 100%。
3. geometry 一致率 100%。
4. label 合法率 100%。
5. hard reject rate 目标不超过 5%。
6. unresolved manual review 为 0。
7. accepted synthetic 第一轮不超过真实训练病例 25%。
8. 每个 source 默认最多 accepted 1 例。
9. tiny/small/large 分布没有明显失控。
10. RC synthetic 只来自真实 RC 语境。
11. 通过 nnU-Net integrity check。
12. real+synth 消融不降低核心真实验证指标。

## 8. G2 输出给 S1/S2

只输出 accepted 或 ablation_only 数据：

```text
work_space/G2/results/nnunet_raw/
  Dataset261_BraTS2026_MET_RealSynth_G1Smoke/
  Dataset262_BraTS2026_MET_RealSynth_G1V1/
```

通道固定：

```text
0000 = t1n
0001 = t1c
0002 = t2w
0003 = t2f
```

validation fold 只能是真实病例，synthetic 不进入 validation。

物化入口：

```bash
python work_space/G2/code/g2_materialize_nnunet_dataset.py \
  --output-root /path/to/nnUNet_raw \
  --synthetic-accepted-manifest work_space/G2/results/manifests/synthetic_accepted_manifest_run_id.csv \
  --dataset-id 261 \
  --dataset-name BraTS2026_MET_RealSynth_G1V1 \
  --channel-order g2_official \
  --mode manifest-only
```

如果 S1/S2 暂时沿用当前代码中的通道顺序，应把 `--channel-order` 改为 `s2_current`，并在训练报告里写清楚。

## 9. G2 回传给 G1

每批 QC 后，G2 回传：

1. accepted case 数。
2. rejected case 数。
3. needs_regeneration case 数。
4. reject reason 分布。
5. ROI boundary 问题样例。
6. z continuity 问题样例。
7. modality consistency 问题样例。
8. source/seed/checkpoint 缺失问题。
9. 建议 G1 下轮调整项。

## 10. 最终报告结构

每批报告必须包含：

1. run 基本信息。
2. G1 checkpoint 与参数。
3. source CSV 版本。
4. raw output 数量。
5. G2 intake 结果。
6. 自动 QC 结果。
7. 人工复查结果。
8. accepted/rejected/needs_regeneration 清单。
9. 扩散生成质量专项结论。
10. nnU-Net integrity 结果。
11. real-only vs real+synth 消融结果。
12. 是否进入主训练的最终结论。

训练后官方指标字段校验入口：

```bash
python work_space/G2/code/g2_official_mets_metrics_parser.py validate-csv \
  --csv-path official_metrics_fold0.csv
```
