## ECNU_EYU
# 资料区（data space）：往年文章和code，2026年task1以及对应的数据集，BIOS的11篇课程学习pdf，每个人的任务安排等等所有参考学习资料；
# 工作区（work space）：会划分每个人的工作区，S1，S2，S3，S4，G1，G2，每个工作区会有一份md文档，来描述自己的整体任务和目前的任务进度；每个人仅仅只能在自己的工作区内修改，上传代码和文件，切勿改动其他同学的内容；
# 时间进度线（time pipeline）：这里会定下阶段性的任务，根据比赛时间节点来严格制定，每个人提交成功之后来这个文档更新自己的任务进度；
# rules：
1:每次提交成果的git comments内容必须用简体中文简单描述自己的改动部分；
2:提交的代码需要具有可读性，有注释或者相关文档说明；
3:提交上传的文档，必须可读，不要出现乱码，公式不要以latex格式呈现；
4:再次提醒，不要改动其他同学工作区的内容；
5:不要提交冗余文件，比如缓存，测试文件造成代码冗余。
6:提交改动尽量以一周两次左右为宜，不低于两次。
